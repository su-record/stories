# fallingo 개발일지 - c969850..748f797 (23개 커밋)

안녕하세요, Fallingo 프로젝트를 개발하는 Su입니다! 🙋‍♂️
벌써 11월의 마지막 날이네요. 베타 런칭 목표인 2025년 12월까지 한 달도 채 남지 않아 매일매일이 바쁘게 흘러가고 있습니다.

이번 개발 일지는 지난 11월 29일부터 오늘(11월 30일)까지 이틀간 진행된 총 23개의 커밋 내역을 바탕으로 작성했어요. 짧은 기간이었지만, 정말 많은 변화가 있었답니다. 주로 백엔드 API와의 연동을 강화하고, 사용자 경험을 개선하기 위한 작업들이 많았습니다.

**작업 기간**: 2025-11-29 ~ 2025-11-30

## 📝 이번 기간 작업 내용

이번 기간 동안 Fallingo 앱의 안정성과 사용성을 높이기 위한 다양한 작업들을 진행했어요. 특히 백엔드 API와의 스키마 정합성을 맞추는 데 많은 시간을 보냈습니다.

### 🧩 API 연동 및 스키마 정합성 확보

백엔드를 FastAPI로 개발하면서 계속해서 API 스키마가 변경되거나 확정되는 부분이 있었어요. 이 과정에서 프론트엔드 Flutter 앱의 모델들도 백엔드 스키마에 맞춰 계속 수정해야 했습니다. 😅

*   **프론트엔드 모델 업데이트**: `refactor: 프론트엔드 모델을 백엔드 API 스키마에 맞춤`, `fix: 피드 모델 API 스키마 맞춤`, `fix: 프론트엔드 모델을 백엔드 API 스키마에 맞게 수정` 등 백엔드 변경 사항에 맞춰 여러 모델을 수정했습니다.
*   **API 버저닝 적용**: API 호출 경로에 `v1` 버저닝을 통합하고, `baseUrl` 설정을 일관되게 맞췄습니다. `fix: API baseUrl /api → /api/v1 수정`, `refactor: API 호출 경로에 v1 버저닝 통합` 커밋들이 대표적입니다.
*   **API 클라이언트 수정**: `camera_screen`에서 `Dio` 사용 시 `baseUrl`이 누락되어 있던 부분을 수정했어요. (`fix: camera_screen에서 baseUrl 없는 Dio 사용 수정`)
*   **작업량**: 백엔드 API와의 호환성을 위한 7개 이상의 주요 모델 및 API 경로 수정 작업을 진행했습니다.

### 🎨 UI/UX 개선 및 리팩토링

사용자에게 더 좋은 경험을 제공하고, 앱의 유지보수성을 높이기 위한 UI/UX 작업들도 활발하게 이루어졌습니다.

*   **Material 3 디자인 적용**: Flutter 3의 Material 3 디자인 시스템을 적용하여 앱의 전반적인 UI를 현대화했습니다. 🎨 아직은 어색한 부분도 있지만, 최신 디자인 가이드를 따르는 것이 장기적으로 좋다고 판단했습니다. `Refactor UI to Material 3, restore legacy themes for stability, remove dark mode` 커밋이 그 결과입니다.
*   **성능 개선**: `MainScreen`에 Lazy Loading을 적용하여 앱 초기 로딩 속도를 개선했습니다. (`perf: MainScreen Lazy Loading 적용`) 쾌적한 사용자 경험을 위해 성능 최적화는 늘 중요하죠!
*   **네비게이션 및 프로필 이미지**: 네비게이션 바에 사용자 프로필 이미지를 표시하는 기능을 추가하고, 그동안 "구현 예정"으로 비워두었던 네비게이션 경로들을 실제 화면과 연결했습니다. (`feat: 네비게이션 바에 프로필 이미지 표시`, `fix: '구현 예정' 플레이스홀더 제거 및 실제 네비게이션 연결`)
*   **지도 화면 리팩토링**: `FF 원칙`을 적용하여 지도 화면을 리팩토링했습니다. (`refactor(map): FF 원칙 적용 - 지도 화면 리팩토링`) `FF 원칙`은 제가 생각하는 Flutter 개발 원칙인데, 나중에 기회가 되면 블로그에 자세히 공유해볼까 해요! 🗺️
*   **작업량**: Material 3 적용 포함 5개 주요 화면 UI 리팩토링, 1개 성능 개선 작업을 완료했습니다.

### 🌐 다국어(i18n) 처리

Fallingo가 더 많은 사용자들에게 다가갈 수 있도록 다국어 처리에 힘썼습니다.

*   **하드코딩 문자열 제거**: 프로필, 설정, 피드 화면 등 여러 곳에 하드코딩되어 있던 한국어 문자열들을 다국어 처리 리소스 파일로 옮겼습니다. (`feat(i18n): 프로필 화면 다국어 처리`, `feat(i18n): 설정/피드 화면 다국어 처리`, `refactor(i18n): 하드코딩 한국어 문자열 다국어 처리`, `refactor(i18n): 하드코딩 한국어 문자열 다국어 처리 (2차)`)
*   **다국어 변수 오류 수정**: `my_markers_screen.dart`에서 다국어 변수가 누락된 부분을 수정했습니다. (`fix: my_markers_screen.dart l10n 변수 누락 오류 수정`)
*   **작업량**: 4개 이상의 핵심 화면에 대한 다국어 처리를 완료했습니다.

## 💡 작업 하이라이트

이번 기간의 가장 큰 하이라이트는 아무래도 **백엔드 API 스키마와 프론트엔드 모델의 정합성을 확보**한 것과 **Material 3 UI 적용 및 MainScreen Lazy Loading**이 아닐까 싶어요.

FastAPI로 백엔드를 만들고, Flutter로 프론트엔드를 만들다 보니 두 시스템 간의 데이터 규약을 맞추는 게 정말 중요하더라고요. 특히 `v1` 버저닝을 도입하고, 그에 맞춰 모든 API 호출 경로와 모델을 수정한 것이 큰 작업이었습니다. 처음에는 API 스펙이 계속 바뀌는 게 당황스러웠지만, 결국 안정적인 통신을 위한 과정이라고 생각하며 끈기 있게 따라갔습니다. 덕분에 앞으로는 API 관련해서 좀 더 안정적으로 개발할 수 있을 것 같아요! 🤝

Material 3로 UI를 전면 개편한 것도 큰 변화였습니다. 기존 Material 2 테마와 Dark Mode를 제거하고 M3를 적용하면서 코드도 많이 바뀌고 시행착오도 겪었지만, 앱의 전반적인 룩앤필이 훨씬 세련되어졌습니다. 메인 화면 Lazy Loading으로 앱이 더 빠르게 뜨는 것도 체감돼서 뿌듯하네요. ✨ 사용자 경험 개선에 직접적으로 기여한 것 같아 기쁩니다.

다수의 `fix: 프론트엔드 모델을 백엔드 API 스키마에 맞게 수정` 커밋이 반복된 것은, 역시나 백엔드 API의 초기 설계가 확정되지 않은 상태에서 프론트엔드 개발이 진행되었기 때문입니다. 또는, 제가 FastAPI를 학습하며 스키마를 최적화하는 과정에서 변화가 잦았던 탓도 있고요. 하지만 이렇게 매번 맞춰나가면서 Fallingo의 핵심 기능들이 점차 단단해지고 있다고 생각합니다! 💪
