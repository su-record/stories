# vibe 개발일지 #53 - 증거 계약(Evidence Contract)과 실행 패킷 컴파일러 도입 (12개 커밋)

**작업 기간**: 2026-07-08 ~ 2026-07-22

안녕하세요, Su입니다! 

AI 코딩 하네스 도구 `vibe`(@su-record/vibe)를 만들면서 항상 가슴속에 품고 있는 철학이 있습니다. **"Agent = Model + Harness"**라는 명제입니다. 모델이 아무리 똑똑해져도, 하네스(Harness)가 실행 환경을 엄격히 통제하고 최소한의 품질을 보장해주지 않으면 "환각(Hallucination)에 기반한 코드 작성"을 막기 어렵습니다.

이번 2주간은 v3.2.0 구조 재편을 바탕으로, AI가 작성한 결과물을 더욱 철저히 검증하는 **'증거 계약(Evidence Contract)'**과 **'실행 패킷 컴파일러'**를 도입했습니다. v3.2.0에서 v3.2.2로 이어지는 12개 커밋의 여정을 공유합니다!

---

## 📝 이번 기간 작업 내용

### 1. 하네스 검증 엔진 고도화 (증거 계약 & 실행 패킷)
- **증거 계약(Evidence Contract) 및 Judge 권한 도입**: AI가 단순히 "수정 완료했습니다"라고 말하는 것을 신뢰하지 않고, 스냅샷·테스트 통과 로그 등 물리적 증거를 제출하도록 강제하는 수치 기반 검증 레이어를 도입했습니다.
- **SPEC 기반 실행 패킷 컴파일러 구현**: `/vibe.spec` 작성 시, AI가 즉시 실행할 수 있는 무결성 검증 형태의 'Execution Packet'으로 컴파일하는 기능을 추가했습니다.
- **결정론적 게이트(Deterministic Gate) 안정화**: 크로스 플랫폼(Windows/macOS) 환경에서 발생하던 테스트 수 카운트 드리프트(Count drift) 및 20/50 규칙 충돌 이슈를 해결했습니다.

### 2. DX 및 스킬 시스템 구조화
- **내장 스킬 네임스페이스 통일**: 모든 내장 스킬을 `vibe.*` 네임스페이스(예: `vibe.build`, `vibe.test` 등)로 일괄 변경하여 가독성을 높이고 네이밍 충돌을 방지했습니다.
- **큐레이션 기반 릴리즈 노트 자동 생성**: 커밋 로그와 SPEC 변경 사항을 AI가 자동 분석하여, 사용자 관점의 가치 중심 릴리즈 노트를 큐레이션하는 기능을 추가했습니다.

### 3. 글로벌 문서 동기화
- **README 영어 지원 강화**: 해외 사용자를 위한 English README 상단 섹션을 추가하고, 지원 도구 표(Claude Code, Codex CLI, Gemini CLI)의 마크다운 링크 표기를 한국어판과 일치시켰습니다.

## 💡 작업 하이라이트

### "AI의 말 대신 증거를 믿는다" — Execution Packet & Judge 시스템

기존에는 AI 에이전트가 코드를 수정하고 "테스트 통과했습니다!"라고 하면 하네스가 이를 그대로 믿는 허점이 있었습니다. 하지만 AI도 가끔 테스트 코드를 주석 처리하거나 단순 모킹으로 넘어가 버리는 편법을 씁니다.

이를 막기 위해 **실행 패킷 컴파일러(Execution Packet Compiler)**와 **Judge 권한**을 도입했습니다.

```
[Before]
/vibe.spec ➔ LLM 코드 작성 ➔ "완료했습니다!" (거짓 보고 가능성 존재)

[After]
/vibe.spec ➔ [Compiler] ➔ Verified Execution Packet 
                           ↓
                   LLM 코드 작성 및 실행
                           ↓
                 [Evidence] 증거 제출 (로그/스냅샷)
                           ↓
                 [Judge] 하네스 엄격 검증 ➔ Done!
```

이제 `/vibe.spec` 문서는 즉시 컴파일되어 검증 파이프라인 형태의 '실행 패킷'이 됩니다. 에이전트는 이 패킷에 정의된 증거 계약(Evidence Contract)을 충족해야만 작업 완수 승인(Judge)을 받을 수 있습니다.

이 과정에서 Cross-Platform OS 간 세퍼레이터 차이로 결정론적 게이트가 깨지는 문제가 있었는데, 이를 수정한 뒤 하네스의 검증 안정성이 훨씬 높아졌습니다.
