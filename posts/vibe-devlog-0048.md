# vibe 개발일지 #48 - 루프 네이티브 엔진과 YAGNI 가드레일 도입 (11개 커밋)

**작업 기간**: 2026-06-11 ~ 2026-06-26

안녕하세요, `vibe` 패키지를 개발하고 있는 10년 차 프론트엔드 개발자 Su입니다. 

"Agent = Model + Harness"라는 철학 아래, 어떻게 하면 AI가 작성하는 코드의 품질을 자동으로 보장하고 개발자가 'Vibe Coding(편안하게 직관으로 개발하는 것)'에만 집중하게 만들 수 있을지 매일 고민하고 있습니다.

지난 2주 동안 `vibe` 프로젝트에는 무려 3번의 마이너/패치 버전 릴리즈가 있었고, AI 코딩 하네스로서의 완성도를 한 단계 끌어올린 중요한 아키텍처적 변화들이 있었습니다. 11개의 커밋을 중심으로 그 치열했던 여정을 공유합니다.

---

## 📝 이번 기간 작업 내용

이번 작업은 크게 **루프 엔진 고도화**, **하네스 스킬 추가**, 그리고 **안정성 강화(Remediation)** 세 가지 영역으로 나누어 진행되었습니다.

### 1. 루프 엔지니어링 & 자동화 레이어 구축 (v2.14.0)
*   **`feat(loop): vibe.loop — loop engineering layer`**: AI가 한 번에 완벽한 코드를 짜지 못할 때, 스스로 피드백을 받아 코드를 수정하고 검증을 통과할 때까지 반복 수행하는 루프 자동화 레이어를 설계했습니다.
*   **`feat(loop): loop-native default`**: 이제 `/vibe` 명령어는 단 한 번의 SPEC 승인만 떨어지면, 하네스의 품질 게이트(테스트, 린트 등)를 모두 통과할 때까지 스스로 내부 루프를 돕니다. 개발자가 중간에 개입할 필요가 없어졌습니다.

### 2. 가드레일 및 스킬 시스템 확장 (v2.15.0 ~ v2.16.0)
*   **`feat(skills): add yagni-ladder core skill`**: AI가 성급하게 미래를 예측해 불필요한 보일러플레이트나 오버엔지니어링 코드를 짜는 것을 차단하는 `YAGNI(You Aren't Gonna Need It)` 가드레일 스킬을 추가했습니다.
*   **`feat(skills): add presentation builder skill`**: 작업 결과를 시각적으로 깔끔하게 정리하여 보고하거나 공유할 수 있는 발표 자료 빌더 스킬을 탑재했습니다.

### 3. 노이즈 제거 및 자가 치유(Remediation) (v2.14.1)
*   **`chore: untrack .vibe/scope.json`**: 매 세션마다 SPECs로부터 자동 도출되는 컨텍스트 스코프 파일이 Git 이력에 지저분하게 남는 현상을 해결했습니다. (생성 시간 기준 변경으로 인한 무의미한 Git Churn 방지)
*   **`fix: 분석 리포트 P1·P2 일괄 보완`**: 6월 11일 자체 분석 리포트에서 발견된 우선순위 P1, P2 결함 5건을 일괄 해결하고 관련 평가 리포트(Evaluation Reports)를 리포지토리에 기록했습니다.

## 💡 작업 하이라이트

### 1. 루프 네이티브(Loop-Native) 워크플로우: "승인은 한 번만, 게이트 통과는 끝까지"
이전의 `vibe`는 `/vibe.spec`으로 명세서를 만들고, `/vibe.run`으로 코드를 실행한 뒤 에러가 나면 개발자가 다시 피드백을 주어야 했습니다. 

하지만 진정한 '하네스'라면 스스로 문제를 진단하고 고쳐야 합니다. 이번에 도입한 **`loop-native`** 기능은 이를 완벽히 자동화합니다.

```
[개발자 SPEC 승인] ➔ [/vibe 루프 시작] ➔ [코드 생성] ➔ [품질 게이트 검증]
                                 ▲                          │ (Fail)
                                 └───── [자가 진단 및 수정] ◄┘
                                                            │ (Pass)
                                                       [최종 완료 (Done)]
```

이제 개발자가 `/vibe`를 실행하고 SPEC을 승인하면, 하네스가 코드를 빌드하고 테스트를 돌려 실패할 경우 스스로 프롬프트를 재구성하여 에러를 수정합니다. 이 과정을 게이트가 완전히 열릴 때까지 무한히(혹은 지정된 임계값까지) 반복합니다. 진정한 "Easy vibe coding + Minimum quality guaranteed"의 실현입니다.

### 2. YAGNI-Ladder 스킬: AI의 오버엔지니어링 제어하기
AI 모델(Claude, Gemini 등)에게 코딩을 시켜보면 가끔 너무 열정이 넘쳐서 탈입니다. "나중에 필요할지도 모른다"며 요구사항에도 없는 추상 클래스, 복잡한 제네릭, 미래용 헬퍼 함수들을 마구 만들어내곤 하죠. 이는 고스란히 기술 부채가 됩니다.

이번에 추가된 **`yagni-ladder core skill`**은 AI가 성급하게 코드를 생성하는 것을 물리적으로 차단합니다. 
*   현재 SPEC에 명시된 요구사항만을 충족하는지 검증합니다.
*   "미래를 대비한 예측성 코드"가 발견되면 하네스가 스스로 "이 코드는 지금 필요하지 않습니다(YAGNI)"라며 거부(Reject) 피드백을 줍니다.

이를 통해 코드베이스는 더 가볍고 명확하게 유지될 수 있게 되었습니다.

## 🤕 아픔과 배움: 무의미한 Git 커밋 지옥(`scope.json`) 탈출기

중간에 꽤 번거로운 문제가 있었습니다. `.vibe/scope.json` 파일은 세션마다 현재 작업 범위를 추적하기 위해 자동으로 생성되는 유틸리티성 메타데이터입니다. 

그런데 이 파일 안에 `generatedAt` 같은 타임스탬프 값이 들어있다 보니, 코드는 전혀 변하지 않았는데도 `/vibe`를 실행할 때마다 Git 변경사항(Churn)이 발생해 Pull Request가 지저분해지는 치명적인 DX(Developer Experience) 문제가 있었습니다.

**"자동 생성 파일은 자동 생성 파일답게 관리하자."**

과감히 이 파일을 `.gitignore`에 추가하고 Git 추적을 중단(`untrack`)했습니다. 대신 각 세션 실행 시 SPECs로부터 완전히 멱등(Idempotent)하게 온메모리 기반으로 유도(Auto-derived)되도록 흐름을 바꿨습니다. 덕분에 Git 커밋 이력이 아주 쾌청해졌습니다. 역시 귀찮은 수작업이나 불필요한 노이즈는 도구 수준에서 제거해 주는 것이 맞습니다.
