## 📝 이번 기간 작업 내용

이번 커밋들은 크게 **1) 핵심 AI 성능 개선, 2) 프론트엔드 UX 흐름 최적화, 3) 주요 버그 및 비즈니스 로직 수정** 세 가지 영역으로 그룹화할 수 있습니다.

### 1. 🚀 핵심 AI 성능 및 인프라 변경 (1개 커밋)

Fallingo는 사용자가 사진을 찍거나 업로드하는 즉시 어떤 음식인지 인식하는 기능이 핵심입니다. 이 속도가 서비스의 성공을 좌우한다고 판단하여, 과감한 엔진 전환을 시도했습니다.

| 커밋 내용 | 설명 |
| :--- | :--- |
| `perf: 실시간 음식 인식을 GPT Vision → Azure Vision 4.0으로 전환 (5~10배 속도 향상)` | 기존에 테스트용으로 사용하던 GPT-4 Vision API는 정확도는 높았지만, 실시간 환경에서 평균 응답 속도가 3초~5초로 너무 느렸습니다. 이를 Azure AI Services의 Vision 4.0(Enhanced features)으로 전환하면서 응답 속도가 5~10배가량 빨라졌습니다. |

| :--- | :--- |

## 💡 작업 하이라이트: AI 엔진 교체 (GPT → Azure)

이번 기간 가장 큰 도전이자 성과는 바로 **실시간 음식 인식 엔진을 교체**한 것입니다.

비전공자로 개발을 시작했고, 최근 FastAPI와 AI 관련 기술을 학습하면서 GPT Vision이 가장 쉽게 접근할 수 있는 솔루션이었습니다. 하지만 아무리 정확해도 속도가 느리면 사용성이 0에 수렴한다는 것을 깨달았습니다.

| 모델 | 평균 응답 시간 (ms) | 개발 비용 효율 | 비고 |
| :--- | :--- | :--- | :--- |
| GPT-4 Vision | 3,000 ~ 5,000 | 낮음 | 정확도는 높으나 지연 시간이 길어 모바일 환경 부적합 |
| Azure Vision 4.0 | 300 ~ 500 | 높음 | 실시간 처리에 최적화, 인식 성능과 속도의 균형점 |

AI 엔진을 Azure로 전환하는 작업은 단순한 API 키 교체가 아니라, 이미지 전처리 파이프라인과 응답 구조를 모두 바꿔야 하는 대공사였습니다. 약 이틀간 관련 문서를 붙잡고 씨름한 결과, 드디어 500ms 이내의 쾌적한 음식 인식 환경을 구축할 수 있었습니다. 사용자 경험이 비약적으로 향상될 것 같아 뿌듯합니다. 😊
