## vica 개발일지: 선택은 더 편하게, 판정은 더 단단하게

이번 작업은 크게 두 갈래였습니다. 하나는 사용자가 터미널에서 승인이나 스펙 선택을 더 편하게 할 수 있도록 인터랙션을 정리한 것이고, 다른 하나는 텍스트를 해석하고 보여주는 과정에서 어색하게 새던 부분들을 다듬은 것입니다. 겉으로는 작은 수정처럼 보일 수 있지만, 실제로는 vica를 “계속 쓰고 싶은 도구”에 가깝게 만드는 작업이었습니다.

### 터미널 선택 경험 개선

기존에는 SPEC이나 승인 단계에서 사용자가 직접 입력해야 하는 흐름이 많았습니다. 기능 자체는 동작했지만, 매번 타이핑하는 방식은 생각보다 피로도가 컸습니다. 특히 개발 흐름 중간에 잠깐 확인만 하고 넘어가고 싶은 상황에서는 키보드 입력 하나하나가 마찰로 느껴졌습니다.

그래서 이번에는 터미널을 raw-mode로 전환해 화살표 키와 핫키로 선택할 수 있도록 바꿨습니다. 이제 사용자는 항목을 위아래로 이동하면서 고르고, 필요한 경우 단축키로 바로 승인하거나 취소할 수 있습니다. 단순히 “입력 방식이 바뀌었다”기보다는, CLI 도구가 조금 더 앱처럼 반응하도록 만든 느낌입니다.

작업하면서 중복 렌더링 문제도 같이 정리했습니다. 선택 화면이 갱신될 때 같은 내용이 여러 번 찍히면 사용자는 현재 상태를 신뢰하기 어렵습니다. 터미널 UI는 화면이 작고 로그가 계속 쌓이기 때문에, 렌더링이 깔끔하지 않으면 금방 산만해집니다. 이번 정리로 선택 상태가 더 또렷하게 보이고, 조작 결과도 예측 가능해졌습니다.

### 마크다운 출력 일관성 정리

또 하나 신경 쓴 부분은 빌드 나레이션 출력입니다. 빌드 과정에서 설명 문장을 보여줄 때 마크다운 문법이 그대로 노출되는 문제가 있었습니다. 예를 들어 줄바꿈이 의도대로 보이지 않거나, 굵게 표시하려던 문법이나 인라인 코드 표시가 원문 그대로 튀어나오는 식이었습니다.

이런 문제는 기능적으로 치명적이지는 않지만, 도구의 완성도를 많이 깎아 먹습니다. 특히 vica처럼 개발 과정을 설명하고 안내하는 도구에서는 메시지가 곧 인터페이스입니다. 사용자가 읽는 문장이 어색하면, 내부 로직이 잘 돌아가고 있어도 전체 경험은 덜 다듬어진 것처럼 느껴집니다.

그래서 빌드 나레이션도 마크다운 렌더 흐름을 타도록 정리했습니다. 줄바꿈은 줄바꿈답게 보이고, 강조나 코드 표기도 의도에 맞게 표현되도록 맞췄습니다. 덕분에 출력이 조금 더 문서처럼 읽히고, 로그가 아니라 안내문에 가까워졌습니다.

### rigor 판정 보강

마지막으로는 rigor 쪽의 어형 처리 문제를 고쳤습니다. 특히 “만든다” 같은 표현이 의도와 다르게 light 쪽으로 새는 문제가 있었습니다. 이건 단순 키워드 매칭만으로는 놓치기 쉬운 부분입니다. 한국어는 어미와 활용에 따라 같은 의미도 다른 표면형으로 나타나기 때문에, 판정 기준이 조금만 느슨해도 엉뚱한 분류가 나올 수 있습니다.

이번 수정에서는 rigor의 바닥 기준을 어형에 더 견고하게 맞췄습니다. 표현이 조금 달라져도 핵심 의도가 흔들리지 않도록 다듬은 셈입니다. 이런 류의 수정은 테스트할 때는 작아 보이지만, 실제 사용 중에는 신뢰도에 직접 영향을 줍니다. “왜 이게 이렇게 분류되지?”라는 순간이 줄어드는 것이 목표였습니다.

### 마무리

이번 변경들은 모두 vica의 기본기를 다지는 작업이었습니다. 선택 UI는 더 편해졌고, 출력은 더 읽기 좋아졌으며, 텍스트 판정은 더 단단해졌습니다. 화려한 신규 기능은 아니지만, 매일 사용하는 도구라면 이런 작은 마찰을 줄이는 일이 결국 가장 중요하다고 느꼈습니다.

다음에는 이 개선된 흐름 위에서 더 복잡한 승인 시나리오나 나레이션 품질을 다듬어볼 수 있을 것 같습니다. 이번 작업을 통해 vica가 조금 더 차분하고 믿을 만한 CLI 도구에 가까워졌습니다.
