## 이번 주 요약

Fallingo 프로젝트의 초기 아키텍처를 완성했다. Phase 3-4 개발을 통해 데이터베이스 스키마, 핵심 도메인 모델, 그리고 기본적인 API 구조를 잡았다. 총 25개의 커밋으로 프로젝트의 기반을 다졌다.

## 주요 작업

### 시스템 설계 완료
- Phase 1-4 단계별 개발 완료
- Step 2-4 세부 구현 완성
- 데이터베이스 스키마 설계
- 도메인 모델 정의

### 기술 스택 결정
- Backend: FastAPI (Python 3.11)
- Database: PostgreSQL 16 + PostGIS 3.3
- Cloud: Google Cloud Platform
- 위치 기반 서비스를 위한 PostGIS 확장

### 핵심 도메인 설계
- User (사용자 관리)
- Feed (음식 피드)
- Restaurant (음식점 정보)
- Follow (팔로우 시스템)
- Point & Tier (포인트/등급 시스템)

## 배운 점

**공간을 다루는 법**

PostGIS로 공간 데이터를 다루는 법을 배웠다. 반경 기반 검색, 거리 계산, 인덱스 전략. 티어에 따라 다른 상호작용 반경을 설계하면서, 위치 기반 서비스의 복잡함을 알게 되었다. 데이터베이스가 단순히 데이터를 저장하는 것이 아니라, 공간을 이해한다는 것이 신기했다.

**FastAPI라는 선택**

Flask 대신 FastAPI를 선택한 이유는 명확했다. 자동 문서화, 타입 힌트 기반 검증, 비동기 처리 지원. Python의 ML/AI 라이브러리를 활용할 수 있다는 점도 마음에 들었다. 개발 속도가 빨라지는 것을 체감했다.

**25개의 커밋, 하나의 기반**

Phase 0부터 2까지, 25개의 커밋으로 프로젝트의 기반을 다졌다. 데이터베이스 스키마, 도메인 모델, API 구조. 아직 화면은 없지만, 탄탄한 백엔드가 만들어지고 있다는 느낌이 들었다. Claude와 함께 체계적으로 설계할 수 있었다.
