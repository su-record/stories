안녕하세요, Fallingo 개발자 Su입니다! 👋

벌써 2026년 새해를 맞이하고, Fallingo 개발도 박차를 가하고 있습니다. 2025년 연말은 다들 잘 보내셨나요? 저는 연휴 기간 동안 미뤄뒀던 핵심 기능의 디테일을 잡고, 특히 최근 집중하고 있는 AI 기반 개발 환경을 확실하게 구축하는 데 시간을 투자했습니다.

이번 개발 일지는 2025년 12월 25일부터 2026년 1월 2일까지의 총 10개 커밋 내역을 정리했습니다. 연말이라 큰 기능을 공격적으로 추가하기보다는, 기존 기능의 완성도를 높이고, AI와의 협업 환경을 다듬는 데 집중했습니다.

## 📝 이번 기간 작업 내용

이번 기간의 커밋들은 크게 네 가지 영역으로 분류할 수 있습니다.

| 영역 | 주요 작업 내역 | 완성도/성과 |
| :--- | :--- | :--- |
| **I. AI/개발 환경 구축** | AI 모델 변경, 프롬프트 템플릿 및 코딩 규칙 추가 | 2가지 모델(Gemini, Claude) 연동 환경 구축 |
| **II. 핵심 기능 및 성능 개선** | 피드 쿼리 최적화, 갤러리 이미지 업로드 (EXIF 검증) | 성능 병목 구간 1곳 해소, 핵심 기능 1개 완성 |
| **III. 사용자 인터페이스 (FE)** | 마이페이지 프로필 통계 및 피드 리스트 개선 | 사용자 정보 제공 범위 확대 |
| **IV. 서비스 정의 및 문서화** | `about.html` 서비스 설명 및 용어 통일 | 잦은 Fix를 통한 서비스 정의 명확화 |

### I. AI/개발 환경 구축 및 모델 최적화 (3건)

최근 FastAPI를 사용하면서 AI 기반 백엔드 로직 설계에 흥미를 느끼고 있습니다. Next.js와 Vue.js만 다루던 프론트엔드 개발자로서 이 새로운 영역은 정말 흥미진진한 도전이에요!

*   `chore: Gemini 모델을 gemini-flash-latest로 변경`
    *   기존에 사용하던 Gemini 모델을 최신 `gemini-flash-latest`로 변경했습니다. AI 모델은 속도와 비용 효율성이 중요하기 때문에, 가장 적합한 모델을 테스트하고 적용하는 과정이 반복되고 있습니다. 성능 향상을 기대하고 있어요.
*   `feat: Claude Code 슬래시 커맨드 및 개발 규칙 추가`
    *   코드 리뷰와 단순 코딩 작업 보조를 위해 Claude 모델을 도입하고, VS Code 환경에서 슬래시 커맨드를 통해 바로 접근할 수 있도록 환경을 설정했습니다. 개발 규칙을 명확히 정의해서, AI가 생성하는 코드가 Fallingo의 코드 컨벤션을 따르도록 가이드라인을 만들었습니다. 🤖
*   `feat: 폴링고 오피셜 프로필 로고 및 AI 프롬프트 템플릿 추가`
    *   Fallingo의 오피셜 AI 챗봇이 사용할 프로필 이미지와, 주요 답변을 위한 프롬프트 템플릿을 추가했습니다. 사용자 응답의 일관성을 확보하기 위함입니다.

### II. 핵심 기능 및 성능 개선 (3건)

Fallingo는 위치 기반 음식 소셜 플랫폼이기 때문에, 데이터의 정확성과 성능이 매우 중요합니다. 특히 사진 업로드 기능은 핵심 사용자 경험이 시작되는 지점입니다.

*   `feat: 갤러리 이미지 업로드 기능 구현 (EXIF 검증)`
    *   갤러리 이미지 업로드 기능을 구현했습니다. 이때 가장 중요했던 것은 **EXIF(Exchangeable Image File Format) 메타데이터 검증** 로직입니다. Fallingo는 위치 기반 서비스이므로, 사진에 담긴 GPS 정보의 유효성을 검사해야 합니다. EXIF 정보가 없거나 변조된 경우에는 사용자에게 알림을 주는 로직을 함께 구현했습니다. 📸
*   `refactor: 피드 서비스 코드 정리 및 쿼리 최적화`
    *   사용자가 가장 자주 접근하는 피드 서비스의 쿼리 최적화 작업을 진행했습니다. Flutter(FE)와 FastAPI(BE) 간의 데이터 전송량을 줄이고, PostgreSQL에서 발생하는 불필요한 조인(Join)을 제거하여 응답 속도를 개선했습니다.
*   `refactor: 폴링고 오피셜 기능 정리`
    *   관리자 및 오피셜 계정으로 사용될 기능들을 재분류하고 코드를 정리했습니다.

### III. 사용자 인터페이스 개선 (1건)

*   `feat: 마이페이지 프로필 통계 및 피드 리스트 개선`
    *   마이페이지에서 사용자의 통계 정보(예: 올린 리뷰 수, 팔로워 수, 가장 많이 방문한 지역 등)를 한눈에 볼 수 있도록 UI를 개선했습니다.

### IV. 서비스 정의 및 문서화 정교화 (3건)

이번 기간에 유난히 `about.html` 문서 수정 커밋이 많았습니다.

*   `fix: about.html 실시간 카메라 메뉴 인식 설명 정확히 수정`
*   `fix: about.html 카메라 메뉴 인식 설명 수정`
*   `fix: about.html에서 '음식점' 표현을 서비스에 맞게 수정`

새해를 맞아 외부 투자 유치 및 마케팅 자료 준비를 위해 서비스 소개 페이지를 다듬고 있습니다. 이 과정에서 Fallingo의 핵심 가치인 '실시간 카메라 메뉴 인식' 기능에 대한 설명이 모호하거나 과장되지 않도록 꼼꼼하게 다듬어야 했습니다. 또한, Fallingo는 단순히 '음식점'이 아니라 '위치 기반 소셜 플랫폼'이므로, 용어의 통일성 확보도 중요했습니다.

## 💡 작업 하이라이트: EXIF 검증과 AI 협업 프로세스

이번 기간의 가장 큰 성과는 **Fallingo의 핵심 가치를 지탱하는 기반을 다진 것**이라고 생각합니다.

1.  **위치 데이터의 무결성 확보 (EXIF):**
    위치 기반 서비스에서 가장 치명적인 오류는 "잘못된 위치에 게시물이 올라오는 것"입니다. 갤러리 업로드 시 사진의 EXIF 데이터를 신뢰할 수 없는 경우를 미리 검증하고 사용자에게 올바른 위치 정보를 입력하도록 유도하는 것은 데이터 무결성을 위해 필수적인 작업이었습니다. 이 기능을 구현함으로써 Fallingo의 데이터 신뢰도가 크게 높아졌습니다.

2.  **AI 중심 개발 프로세스 구축:**
    10년 이상 프론트엔드 개발을 해왔지만, 최근 AI를 통해 백엔드(FastAPI)와 앱(Flutter) 개발을 시도하면서 개발 방식이 완전히 바뀌고 있습니다.

    | 항목 | 이전 (FE 위주) | 현재 (AI 협업) |
    | :--- | :--- | :--- |
    | **새 기술 학습** | 공식 문서 정독 및 샘플 프로젝트 생성 | Claude/Gemini에게 즉시 질문 및 코드 요청 |
    | **반복 작업** | 직접 스니펫 작성 | 슬래시 커맨드로 AI에 컨벤션 기반 코드 생성 요청 |
    | **코드 품질** | Peer Review | AI Code Review 및 규칙 기반 Refactoring |

    이번 커밋에서 Claude 및 Gemini 모델을 세팅하고 규칙을 추가한 것은, 제가 혼자 모든 것을 하려 하기보다 AI를 유능한 팀원으로 활용하기 위한 정규화 작업이었습니다. 비전공자 출신으로서 새로운 기술을 빠르게 흡수할 수 있도록 도와주는 최고의 파트너입니다.
