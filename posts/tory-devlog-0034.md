# tory 개발일지 #4 - 보안 샌드박스 및 비즈니스 모델 구축 (15개 커밋)

**작업 기간**: 2026-04-07 ~ 2026-04-09

## 📝 이번 기간 작업 내용

### 보안 및 실행 인프라 강화 (4개 커밋)

에이전트가 외부 웹사이트에 접속하거나 코드를 실행할 때 발생할 수 있는 보안 위협을 차단하기 위해 샌드박스 환경을 구축했습니다. Docker 기반의 격리된 인프라를 마련하고, CDP(Chrome DevTools Protocol)를 통한 브라우저 제어 시 엄격한 보안 정책을 적용했습니다. 또한 사용자의 승인이 필요한 동작에 대해 타임아웃 처리와 감사 로그 기능을 추가하여 가시성을 확보했습니다.

| 커밋 | 내용 |
|------|------|
| `feat(security): CDP 보안 정책 + Docker 샌드박스 인프라` | 웹 자동화 도구 실행을 위한 격리 환경 구축 |
| `feat(security): 승인 게이트 강화` | 타임아웃, 감사 로그, 상세 설명 기능을 포함한 승인 절차 고도화 |
| `feat(tools): CDP 브라우저 도구 추가` | 네이버 로그인 및 예약 등 웹 자동화 기능 구현 |
| `feat(core): AppCore에 ConcurrencyGuard 통합` | 핵심 로직 실행 시 동시성 제어 레이어 추가 |

### 비즈니스 모델 및 리소스 제어 (3개 커밋)

서비스 지속 가능성을 위해 Free와 Pro 플랜을 구분하는 과금 게이트를 구현했습니다. 단순히 기능 제한에 그치지 않고, 시스템 리소스를 효율적으로 배분하기 위해 플랜별 동시 실행 제한 로직을 적용했습니다.

| 커밋 | 내용 |
|------|------|
| `feat(billing): 동시 실행 제한` | Free 플랜 1건, Pro 플랜 N건으로 동시 실행 수 차등화 |
| `feat: Pro 게이트 + 로컬 AI 설치 UI` | 유료 사용자 전용 기능 접근 제어 및 로컬 LLM 설치 인터페이스 제공 |
| `feat: 언어학습 Pro 전용 기능 추가` | 특정 비즈니스 도메인(언어학습)의 유료화 적용 |

### 에이전트 지능 및 도구 고도화 (5개 커밋)

에이전트가 스스로의 역할을 정의하고 복잡한 작업을 수행할 수 있도록 지능적인 구조를 추가했습니다. 소크라테스식 인터뷰를 통해 사용자의 의도를 정교화하고 이를 바탕으로 에이전트의 사양(Seed)을 생성하는 Ouroboros 패턴을 도입했습니다. 또한 에러 발생 시 스스로 복구하는 체인과 컨텍스트 압축 기술을 통해 장기 실행 성능을 개선했습니다.

| 커밋 | 내용 |
|------|------|
| `feat(secretary): 소크라테스 인터뷰 + Seed 생성` | Ouroboros 패턴 기반의 에이전트 사양 자동 정의 |
| `feat(skills,tools): Hermes 스타일 스킬 자동생성` | 에이전트가 필요에 따라 스킬을 자동 생성하고 내장 도구 4종 통합 |
| `feat(llm,tools,meeting): 에러 복구 체인` | 도구 실행 실패 시 복구 로직 및 다단계 컨텍스트 압축 적용 |
| `feat: 세션 컴팩션 + Gemma4 번들 설치` | 대화 세션 최적화 및 로컬 AI 모델(Gemma4) 설치 지원 |
| `feat(channels): ChannelBridge 통합 레이어` | Secretary와 외부 채널(Discord 등) 간의 통신 구조 통합 |

Tauri 기반의 데스크톱 앱뿐만 아니라 웹, Discord봇, PWA 등 다양한 접점을 마련했습니다. 또한 개발 효율을 높이기 위해 코드 모듈 분리와 CI 워크플로우를 개선했습니다.

## 💡 작업 하이라이트

**Ouroboros 패턴을 통한 에이전트 자아 정의**

이번 작업에서 가장 핵심적인 부분은 에이전트가 사용자와의 인터뷰를 통해 자신의 역할과 작업 사양을 스스로 생성하는 'Ouroboros 패턴'의 도입입니다. 기존에는 개발자가 에이전트의 페르소나를 직접 코드로 정의해야 했으나, 이제는 소크라테스식 질문 과정을 통해 사용자의 추상적인 요구사항을 구체적인 Seed 스펙으로 전환합니다. 이는 에이전트가 단순한 도구를 넘어 스스로의 업무 범위를 규정하는 자율성을 확보했음을 의미합니다.

**보안 샌드박스와 자동화 도구의 통합**

CDP(Chrome DevTools Protocol)를 이용한 웹 자동화 도구는 강력하지만 보안상 위험이 큽니다. 이를 해결하기 위해 모든 브라우저 자동화 동작을 Docker 샌드박스 내부에서 실행하도록 격리했습니다. 또한 '승인 게이트'를 강화하여 에이전트가 민감한 동작(로그인, 예약 등)을 수행하기 전 반드시 사용자의 명시적인 승인을 기록하고, 모든 과정을 감사 로그(Audit Log)로 남기도록 설계했습니다. 이는 자율형 에이전트의 신뢰성을 담보하는 기술적 기반이 됩니다.

**수익화를 위한 동시성 제어 아키텍처**

비즈니스 모델 구현을 위해 단순한 기능 토글 방식을 넘어, 서버 리소스 점유율을 제어하는 방식을 택했습니다. `ConcurrencyGuard`를 통해 Free 플랜 사용자는 한 번에 하나의 작업만 수행할 수 있도록 제한하고, Pro 사용자는 다수의 에이전트를 동시에 운용할 수 있도록 차등화했습니다. 이는 서비스의 안정적인 운영과 수익화를 동시에 달성하기 위한 구조적 해결책입니다.
