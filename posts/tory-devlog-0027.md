# tory 개발일지 #27 - 회의 실시간 스트리밍 + 인텐트 개선 (10개 커밋)

**작업 기간**: 2026-03-18

## 📝 이번 기간 작업 내용

### CLI 타임아웃 + 모델 지정 (3개 커밋)

CLI에서 LLM 호출 시 발생하던 타임아웃 문제를 단계적으로 수정했다. 처음에는 60초를 120초로 늘렸으나 근본적으로는 타임아웃을 제거하고 프로세스 완료까지 대기하는 방식으로 변경했다. `--model` 플래그를 추가해 CLI 호출 시 사용할 모델을 직접 지정할 수 있게 됐다.

| 커밋 | 내용 |
|------|------|
| `fix(llm): CLI 타임아웃 60초 → 120초로 확장` | 타임아웃 연장 |
| `fix(llm): CLI 타임아웃 제거 — 프로세스 완료까지 대기` | 타임아웃 제거 |
| `feat(llm): CLI 호출 시 --model 플래그로 모델 지정` | 모델 지정 플래그 |

### 인텐트 분류 개선 (2개 커밋)

Secretary의 인텐트 분류 로직을 개선했다. 회의 모드를 최우선으로 판단하고 불확실한 경우 LLM이 2차로 판단하는 구조로 변경했다. 회의 모드로 진입했을 때 head의 즉답을 사용자 안내문으로 제한해 회의 결과와 head의 단독 의견이 뒤섞이지 않도록 했다.

| 커밋 | 내용 |
|------|------|
| `fix(secretary): 인텐트 분류 개선 — 회의 최우선 + LLM 2차 판단` | 인텐트 분류 재설계 |
| `fix(secretary): 회의 모드 시 head 즉답을 안내문으로 제한` | 회의 head 응답 제한 |

### 회의 실시간 스트리밍 + 채팅 전면 수정 (5개 커밋)

회의 진행 중 각 발언이 생성되는 대로 실시간으로 표시되는 스트리밍 기능을 구현했다. CLI에서는 stdout 줄 단위로, 데스크톱에서는 Token 이벤트를 통해 실시간 업데이트된다. 회의 채팅 UI를 전면 수정하고 참여자 선정 방식도 프로필(모델) 단위로 재설계했다. 메시지 표시 순서와 중복 문제도 수정했다.

| 커밋 | 내용 |
|------|------|
| `feat: 회의 실시간 스트리밍 — CLI stdout 줄 단위 + Token 이벤트` | 실시간 스트리밍 구현 |
| `fix(desktop): 회의 채팅 실시간 표시 전면 수정` | 채팅 UI 전면 수정 |
| `fix(llm): 참여자 선정을 프로필(모델) 단위로 전면 재설계` | 참여자 선정 재설계 |
| `fix(desktop): 회의 채팅 표시 순서 + 중복 문제 수정` | 순서 및 중복 수정 |
| `fix: 회의 실시간 스트리밍 핵심 버그 3건 수정` | 스트리밍 버그 수정 |

## 💡 작업 하이라이트

**스트리밍은 파이프라인 전체의 문제**

회의 실시간 스트리밍을 구현하는 과정에서 버그가 여러 레이어에 걸쳐 나타났다. CLI stdout 처리, Token 이벤트 발행, 프론트엔드 수신 및 렌더링까지 각 단계가 모두 맞아야 한다. 이번에 핵심 버그 3건을 별도 커밋으로 수정한 것은 각 버그의 원인이 서로 다른 레이어에 있었기 때문이다. 스트리밍은 완성하고 나면 단순해 보이지만 안정화까지 여러 번의 반복이 필요하다.

**참여자를 프로필 단위로**

기존에는 참여자 선정이 모델 이름 기반이었다. 이번에 프로필 단위로 재설계하면서 같은 모델을 쓰더라도 다른 페르소나로 회의에 참여할 수 있게 됐다. 인텐트 분류 개선과 맞물려 회의 소집 판단이 더 정확해졌고, 참여자가 누구인지도 명확해졌다.
