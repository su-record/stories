# tory 개발일지 #17 - 7단계 품질 게이트 (10개 커밋)

**작업 기간**: 2026-03-09

## 📝 이번 기간 작업 내용

### P1 — WCAG P2/P3 접근성 17건 수정 (1개 커밋)

SVG에 aria 속성을 추가하고, 버튼에 type 속성을 명시하고, 슬라이드와 탭 패턴에 ARIA 역할을 부여했다. 이전 기간의 P1 7건에 이어 P2/P3 17건을 추가로 해소했다. 키보드 내비게이션과 스크린 리더 호환성이 개선됐다.

| 커밋 | 내용 |
|------|------|
| `fix: WCAG P2/P3 접근성 17건 수정 — SVG, 버튼, 슬라이드, 탭 패턴` | WCAG P2/P3 접근성 수정 |

### P2 — 파일 시스템 도구 5종 추가 (1개 커밋)

AI 자율 코드 편집을 지원하기 위한 파일 시스템 도구 5종을 추가했다. 읽기, 쓰기, 수정, 삭제, 목록 조회로 구성된다. Closed Loop에서 AI가 코드를 수정할 때 이 도구를 통해 파일에 접근한다. 도구 없이는 LLM이 파일 내용을 추론에만 의존해야 했다.

| 커밋 | 내용 |
|------|------|
| `feat: 파일 시스템 도구 5종 추가 — AI 자율 코드 편집 지원` | 파일 I/O 도구 5종 |

### P3 — Rust unwrap 제거 + 서버 테스트 56개 (1개 커밋)

Rust 서버 코드에서 `unwrap()` 호출 13건을 `?` 연산자로 교체했다. `unwrap()`은 None이나 Err 값을 받으면 패닉을 발생시킨다. 서버 환경에서는 패닉이 곧 프로세스 종료를 의미한다. 서버 테스트도 56개 추가했다.

| 커밋 | 내용 |
|------|------|
| `fix: P1 unwrap() 13건 제거 + 서버 테스트 56개 추가` | unwrap 제거 + 서버 테스트 |

### P4 — rag.ts 모듈 분리 + 웹앱 테스트 기반 (1개 커밋)

단일 파일로 비대해진 rag.ts를 도메인별로 분리했다. 웹앱 테스트 기반도 이 커밋에서 구축했다. 이전까지 웹앱에는 테스트가 없었다.

| 커밋 | 내용 |
|------|------|
| `refactor: P2 — rag.ts 모듈 분리 + 웹앱 테스트 기반 구축` | RAG 모듈 분리 + 웹 테스트 |

### P5 — Biome 린터 도입 + auth-store 테스트 17개 (1개 커밋)

Biome 린터를 도입해 코드 스타일 규칙을 자동화했다. auth-store 테스트 17개를 추가해 인증 로직의 기본 동작을 검증한다.

| 커밋 | 내용 |
|------|------|
| `chore: P3 — Biome 린터 도입 + auth-store 테스트 17개 추가` | Biome 도입 + 인증 테스트 |

### P6 — Biome 경고 44건 해소 (1개 커밋)

Biome 도입 후 경고 44건이 나왔다. `isNaN` 대신 `Number.isNaN`을 써야 하는 버그 6건과 button에 type 속성을 명시해야 하는 경고 34건이 주를 이뤘다. `isNaN`은 문자열을 암묵적으로 숫자로 변환해 예상치 못한 결과를 만들 수 있다.

| 커밋 | 내용 |
|------|------|
| `fix: P4 — Biome 경고 44→0 해소 (isNaN 버그 6건 + button type 34건)` | Biome 경고 전수 해소 |

### P7 — 전체 typecheck 통과 + 서버 테스트 31개 + CI 파이프라인 (3개 커밋)

전체 패키지 typecheck를 통과시켰다. web vitest 타입 문제와 desktop 의존성 문제가 남아있었다. 서버 테스트 31개를 추가로 작성했다. HTML escape, AI 변환, 과금 마크업 로직을 커버한다. PR용 품질 게이트 CI 파이프라인을 추가해 typecheck, lint, test가 PR마다 자동으로 실행된다. 마지막 커밋은 삭제 처리됐다.

| 커밋 | 내용 |

| `fix: P5 — 전체 패키지 typecheck 통과 (web vitest 타입 + desktop 의존성)` | 전 패키지 typecheck 통과 |
| `test: P6 — 서버 테스트 31개 추가 (HTML escape, AI 변환, 과금 마크업)` | 서버 테스트 31개 |
| `ci: P7 — PR용 품질 게이트 CI 파이프라인 추가` | PR 품질 게이트 CI |

## 💡 작업 하이라이트

**7단계 품질 게이트 순서**

이 기간의 작업은 우선순위 순서가 명확했다. P1(WCAG 접근성) → P2(파일 도구) → P3(unwrap 제거) → P4(모듈 분리) → P5(Biome 도입) → P6(경고 해소) → P7(CI) 순서로 진행됐다. 보안, 안정성, 코드 품질, 자동화 순으로 중요도를 정했다.

**Biome 도입 효과**

ESLint 대신 Biome을 선택한 이유는 속도다. Rust로 작성된 Biome은 설정 없이 바로 실행된다. 도입 첫날 경고 44건이 쏟아졌지만 전부 같은 날 해소했다. CI에 통합하면 이후 PR에서 같은 유형의 문제가 자동으로 차단된다.
