## 💡 작업 하이라이트

**companion 모드: 채널이 늘어나면 UI가 달라진다**

Discord 연동 전까지는 데스크톱 채팅 UI가 유일한 입력 채널이었다. Discord 봇이 추가되면 같은 내용을 두 채널에서 받을 수 있고, 데스크톱 UI가 항상 전면에 있을 필요가 없어진다. companion 모드는 이 상황을 위한 UI 상태다. Discord가 연결된 경우 채팅 창을 숨기고 작은 상태 표시만 남긴다. 채널이 다양해질수록 앱의 물리적 존재감은 줄어들어야 한다는 방향성을 담고 있다.

**patch + git 도구: 코딩 에이전트의 핵심 동작**

AI가 코드를 제안할 때 가장 실용적인 형태는 diff다. 전체 파일을 다시 쓰지 않고 변경 부분만 적용할 수 있기 때문이다. `patch` 도구는 unified diff 형식의 입력을 받아 파일에 직접 적용한다. `git` 도구는 상태 확인, 스테이징, 커밋, 로그 조회를 지원한다. 두 도구가 합쳐지면 AI가 변경을 제안하고 검토 후 커밋하는 워크플로우를 에이전트 루프 안에서 완결할 수 있다. closed-loop 크레이트가 실제로 코드를 고치는 사이클을 돌리기 위한 전제 조건이기도 하다.

**프롬프트 캐싱: 반복 컨텍스트 비용 절감**

회의에서 시스템 프롬프트와 SOUL은 매 발언마다 반복 전송된다. 프롬프트 캐싱을 활성화하면 동일한 접두 컨텍스트를 캐시 히트로 처리해 토큰 비용을 줄인다. 회의 fast 모델 기본 설정과 함께 적용해 회의 응답 속도와 비용 효율 양쪽을 개선했다.
