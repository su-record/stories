# fallingo 개발일지 - ffd40da..6fb8dd5 (20개 커밋)

**작업 기간**: 2025-11-21 ~ 2025-11-29

안녕하세요, Fallingo 개발자 Su입니다! 👋

벌써 11월의 마지막 주말이네요. 2025년 12월 베타 런칭 목표를 향해 정말 정신없이 달려온 한 주였습니다. 지난 며칠간 `ffd40da`부터 `6fb8dd5`까지 총 20개의 커밋을 쌓아 올렸는데요, 이번 기간 동안 Fallingo에 어떤 변화들이 있었는지 개발 일지를 통해 공유해 드릴게요.

## 📝 이번 기간 작업 내용

이번 기간에는 특히 사용자 경험(UX) 개선을 위한 핵심 기능 개발과 AI 서비스 고도화, 그리고 개발의 근간이 되는 프론트엔드 아키텍처 및 코드 품질 개선에 많은 시간을 쏟았습니다.

### 1. 코어 프론트엔드 기능 개발 및 사용자 경험 개선 (Flutter 중심)

Fallingo의 얼굴인 프론트엔드 단에서 사용자분들이 더욱 편리하게 서비스를 이용하실 수 있도록 다양한 기능을 추가하고 개선했습니다.

*   **글로벌 지원 강화 🌍**: `feat(i18n): 14개 언어 다국어 문자열 추가` 커밋을 통해 14개 언어를 지원하는 다국어 시스템의 기틀을 마련했어요. Fallingo가 전 세계 맛집을 연결하는 플랫폼이 되기를 바라거든요!
*   **성능 최적화 🚀**: `perf(image): CachedNetworkImage로 이미지 캐싱 적용` 커밋으로 이미지 로딩 속도를 대폭 개선했습니다. 빠릿빠릿한 앱 사용 경험은 기본 중의 기본이죠!
*   **사용자 편의 기능 추가 ✨**:
    *   `feat(permission): 사진 라이브러리 권한 추가`: 사진 업로드 기능의 안정성을 높였어요.
    *   `feat(legal): 법률 문서 시스템 언어 자동 감지`: 사용자의 시스템 언어에 맞춰 법률 문서를 제공하도록 구현했습니다.
    *   `feat(notification): 알림 시스템 구현`: 푸시 알림 등 사용자에게 중요한 정보를 전달할 수 있는 기반을 마련했어요.
*   **핵심 지도/마커 기능 완성 🗺️**:
    *   `feat(map): 유료 마커 공유 지도 및 지오펜싱 지도 구현`
    *   `feat(marker): 유료 마커 시스템 구현`
    유료 마커 시스템과 함께 지오펜싱(Geofencing) 기능을 지도에 적용해서 Fallingo만의 독특한 맛집 경험을 제공할 수 있게 되었어요. Flutter로 지도 기능을 구현하는 게 처음이라 쉽지 않았지만, 구현하고 나니 정말 뿌듯합니다!
*   **추천 맛집 배지 시스템 🏅**: `feat(search): 추천맛집 연도별 배지 시스템 구현`을 통해 사용자들이 맛집에 대한 정보를 한눈에 파악하고, 재미 요소를 더할 수 있도록 준비했습니다.

이번 기간에만 총 **8개 이상의 사용자 지향 핵심 기능**을 추가하고 **이미지 로딩 성능을 획기적으로 개선**했습니다.

### 2. AI 서비스 고도화 및 백엔드 API 안정화 (FastAPI, Gemini)

제가 요즘 집중적으로 학습하고 있는 AI와 FastAPI 관련 작업도 빼놓을 수 없죠! Fallingo의 핵심 경쟁력이 될 AI 추천 서비스를 더욱 똑똑하게 만들었어요.

*   **Gemini 프롬프트 전략 적용 🧠**: `feat: AI 서비스에 Gemini 프롬프트 전략 및 9가지 추론 원칙 적용` 커밋을 통해 AI 추천 서비스의 로직을 한 단계 업그레이드했습니다. 프롬프트 엔지니어링 개념을 깊이 파고들면서 Gemini가 사용자에게 더 정확하고 매력적인 맛집을 추천할 수 있도록 만들었어요. 처음에는 시행착오도 많았지만, 덕분에 AI의 잠재력을 더욱 끌어올릴 수 있었습니다.
*   **AI 엔드포인트 효율화 🔄**: `refactor: AI 검색 제안 엔드포인트 제거 (AI 추천 화면에 통합)` 커밋으로 AI 관련 API를 더 효율적으로 통합했습니다. 중복되는 로직을 줄이고 관리 포인트를 단일화했죠.
*   **API 스키마 및 필터 개선 💪**:
    *   `fix: 피드 상세 API 스키마 불일치 수정 및 좌표 추출 개선`
    *   `refactor: FeedListRequest에서 미사용 tier 필터 제거`
    백엔드 API 스키마의 불일치를 수정하고, 불필요한 필터를 제거하여 데이터의 정합성을 높이고 API 호출의 효율을 개선했습니다.

AI 서비스 관련 기능 **1개**와 백엔드 API **3건의 개선 및 수정**을 완료했습니다.

### 3. 프론트엔드 아키텍처 및 코드 품질 대대적 개선

오랜 기간 개발을 해오면서 '기본기'의 중요성을 항상 느끼고 있어요. 이번 기간에는 Fallingo의 프론트엔드 코드 베이스를 더욱 견고하게 다지는 데 집중했습니다.

*   **Frontend Fundamentals 리팩토링 🏗️**: `refactor: Frontend Fundamentals 원칙 적용 (FF 리팩토링)`과 `refactor: Frontend Fundamentals 10점 만점 수준으로 개선` 커밋은 이번 기간의 가장 큰 작업 중 하나였습니다. 클로드와 함께 진행한 프론트엔드 기초 리뷰 결과를 바탕으로 Fallingo의 프론트엔드 코드 전반에 걸쳐 탄탄한 원칙들을 적용했습니다. 단순히 코드를 고치는 것을 넘어, 아키텍처 레벨에서 견고함을 확보한 작업이었습니다.
*   **코드 품질 개선 (긴 함수 리팩토링) 📉**: `refactor: 코드 품질 개선 - TOP 4 긴 함수 리팩토링 (-54% 라인 수 감소)` 커밋은 저에게도 큰 도전이었는데요, 길고 복잡했던 함수 4개를 집중적으로 리팩토링해서 코드 라인 수를 무려 54%나 줄였습니다. 덕분에 가독성과 유지보수성이 정말 좋아졌고, 앞으로 새로운 기능을 추가하거나 버그를 수정할 때 훨씬 수월할 거예요.

총 **3건의 대규모 프론트엔드 아키텍처 및 코드 품질 개선**을 완료했습니다.
