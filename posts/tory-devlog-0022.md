# tory 개발일지 #22 - 회의→채팅 통합 (10개 커밋)

**작업 기간**: 2026-03-14 ~ 2026-03-15

## 📝 이번 기간 작업 내용

### SSE 타임아웃 + 인텐트 분석 개선 (3개 커밋)

회의 SSE 연결이 장시간 응답 없이 타임아웃되는 문제를 해결했다. idleTimeout을 255초로 설정하고 keep-alive 이벤트를 주기적으로 전송해 연결을 유지한다. 장시간 회의에서 중간에 연결이 끊기는 현상이 사라졌다.

인텐트 분석에서 제안, 아이디어, suggest 같은 키워드가 누락되는 문제를 먼저 패턴 추가로 보강했다. 이어서 인텐트 분석 전체를 LLM 판단으로 전환했다. regex 키워드 매칭은 엣지 케이스가 많고 유지보수가 어렵다. LLM이 문장 전체의 의미를 파악해 인텐트를 분류하는 방식이 더 정확하고 확장성이 있다.

| 커밋 | 내용 |
|------|------|
| `fix: 회의 SSE 타임아웃 해결 — idleTimeout 255초 + keep-alive 이벤트` | SSE 연결 유지 |
| `fix: 인텐트 패턴 보강 — 제안/아이디어/suggest 등 누락 키워드 추가` | 인텐트 키워드 보강 |
| `fix: 인텐트 분석을 LLM 판단으로 전환 — regex 키워드 의존 제거` | 인텐트 LLM 전환 |

### 캔버스 자동 연동 + OAuth + 프로바이더 디스커버리 (2개 커밋)

회의 결과가 완료되면 canvas SSE 이벤트를 통해 캔버스에 자동으로 표시되도록 연동했다. 별도 버튼을 누르지 않아도 회의 결과가 캔버스로 흘러간다. OAuth 로그인 안정화와 통합 프로바이더 디스커버리도 같이 처리했다. 사용 가능한 프로바이더를 런타임에 자동으로 탐색한다.

| 커밋 | 내용 |
|------|------|
| `feat: 회의 결과 캔버스 자동 연동 — SSE canvas 이벤트 추가` | 회의 결과 캔버스 자동 연동 |
| `feat: LLM 기반 회의 트리거 + OAuth 로그인 안정화 + 통합 프로바이더 디스커버리` | 회의 트리거 및 인프라 |

### 회의→채팅 통합 아키텍처 (3개 커밋)

회의 메시지를 별도 UI에서 표시하는 방식을 제거하고 채팅 세션 내 일반 메시지로 통합했다. 사용자 입장에서 회의와 채팅이 같은 공간에서 이어진다. Codex CLI 호출을 수정하고 회의 역할에서 페르소나를 제거하며 모델명 식별 방식을 정리했다.

채팅을 단톡방 아키텍처로 전환했다. 회의는 채팅의 확장이다. 일반 대화에서 특정 조건이 충족되면 자연스럽게 회의 모드로 전환되고, 회의가 끝나면 다시 채팅 흐름으로 돌아온다. 회의 시스템의 별도 진입점을 제거하고 채팅 하나로 통합했다.

| 커밋 | 내용 |
|------|------|
| `fix: CLI workspace 오염 방지 + 타이핑 인디케이터 위치 수정` | CLI 및 UI 수정 |
| `fix: 회의 메시지 별도 UI 제거 — 채팅 세션 내 일반 메시지로 통합` | 회의-채팅 UI 통합 |
| `fix: Codex CLI 호출 수정 + 회의 역할 페르소나 제거 + 모델명 식별` | 회의 역할 및 CLI 수정 |

| 커밋 | 내용 |

| `refactor: 회의 시스템 전면 제거 + 코드 품질 리팩토링` | 분리 회의 코드 전면 제거 |

## 💡 작업 하이라이트

**regex에서 LLM으로**

인텐트 분석의 접근 방식이 바뀌었다. regex 키워드 목록은 작성하기 쉽지만 "이거 어떻게 생각해?"처럼 명시적 키워드 없이 의도가 담긴 문장을 잡지 못한다. LLM이 문장 전체를 읽고 인텐트를 판단하면 이런 엣지 케이스를 자연스럽게 처리한다. 관리할 키워드 목록이 사라지고 판단 로직이 단순해졌다.

**채팅이 회의를 삼키다**

회의를 별도 기능으로 유지하는 것보다 채팅 안으로 통합하는 것이 맞다는 결론에 도달했다. 사용자는 "회의 시작" 버튼을 누르지 않는다. 대화하다가 AI들이 알아서 의견을 모으고, 결론이 나면 채팅 흐름이 이어진다. 단톡방 아키텍처는 이 흐름을 자연스럽게 만든다.
