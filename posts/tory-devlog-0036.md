# tory 개발일지 #12 - 인터랙티브 실행 엔진 고도화 및 도구 생태계 확장 (13개 커밋)

**작업 기간**: 2026-04-10

## 📝 이번 기간 작업 내용

### 멀티모달 인터랙션 및 시스템 최적화 (4개 커밋)

사용자와의 소통 방식을 확장하기 위해 음성 설정 및 자동 번역 체계를 강화했습니다. STT(Speech-to-Text) 인프라와 마이크 설정, 웨이크워드 감지 기능을 데스크톱 클라이언트에 통합했습니다. 특히 DeepL API를 주 번역 수단으로 사용하되, API 장애나 제한 발생 시 LLM이 즉시 대체하는 폴백 체인을 구축하여 번역 업무의 연속성을 보장했습니다. 또한, 사용자 PC의 하드웨어 사양(RAM, VRAM 등)을 자동으로 감지하여 최적의 로컬 LLM 모델을 추천하는 기능을 추가했습니다. 이는 개인정보 보호가 중요한 작업에서 로컬 추론을 유도하기 위한 조치입니다.

| 커밋 | 내용 |
|------|------|
| `feat(desktop): 음성 설정 완성` | STT, 마이크 장치 선택, 웨이크워드 및 테스트 인터페이스 구현 |
| `feat(translation): DeepL API 연동 + LLM 폴백` | DeepL 기반 고속 번역 및 에러 발생 시 LLM 자동 전환 로직 추가 |
| `feat(desktop): PC 사양 감지 → 로컬 LLM 모델 자동 추천` | 시스템 리소스 분석 기반 온디바이스 모델(Ollama/Llama 등) 추천 |
| `fix(accessibility): core-graphics highsierra feature 활성화` | 특정 OS 환경에서의 그래픽 가속 및 접근성 이슈 해결 |

### 콘텐츠 생산 도구 및 실행 파이프라인 (5개 커밋)

Tory의 실행 엔진(Execution Engine)이 수행할 수 있는 도구 범위를 검색과 영상 편집 영역으로 확장했습니다. 네이버 검색 API를 통해 실시간 정보를 수집하고, 음식점의 예약 정보 등 세부 메타데이터를 파싱하는 기능을 구현했습니다. 영상 처리 분야에서는 Whisper STT를 활용한 자막 자동 생성뿐만 아니라, 영상 내 하이라이트를 추출하여 쇼츠(Shorts) 형태로 자동 분할하고 SNS에 업로드하는 파이프라인을 구축했습니다. 이는 에이전트가 단순 정보 전달을 넘어 실제 콘텐츠 제작 업무를 대행할 수 있음을 의미합니다.

| 커밋 | 내용 |
|------|------|
| `feat(tools): 네이버 검색 API + 음식점 예약 정보 수집` | 외부 API 연동을 통한 실시간 정보 획득 도구 확장 |
| `feat(tools): 하이라이트 추출 + 쇼츠 자동 분할` | 영상 분석 기반 구간 추출 및 세로형 영상 재구성 로직 |
| `feat(tools): Whisper STT 기반 자막 자동 생성` | 음성 인식을 통한 정확한 타임코드 자막 생성 기능 |
| `feat(tools): 동영상 편집→SNS 업로드 파이프라인` | 제작된 콘텐츠의 플랫폼 업로드 자동화 프로세스 |
| `feat(glass): 해외사이트 가입/클라우드 설정 단계별 가이드` | 복잡한 웹 UI 상의 절차를 안내하는 오버레이 가이드 시스템 |

### UI/UX 기능 확장 및 코드 베이스 정비 (4개 커밋)

사용자 인터페이스의 유연성을 위해 캔버스 컴포넌트를 고도화했습니다. 프로토타입 요소의 드래그 및 리사이즈 기능을 구현하고, 사진 비교 슬라이더와 문서 편집기를 통합하여 에이전트가 생성한 결과물을 즉시 수정할 수 있게 했습니다. 또한 자율 에이전트의 위험 동작(결제, 데이터 삭제 등)을 방지하기 위해 실행 전 사용자 확인을 거치는 다이얼로그를 도입했습니다. 내부적으로는 디스코드 봇의 호출 구조를 CLI 라이브러리로 위임하여 중복 코드를 제거하고 관리 효율을 높였습니다.

| 커밋 | 내용 |
|------|------|
| `feat(canvas): 프로토타입 드래그/리사이즈 + 사진 비교 슬라이더 + 문서 편집기` | 시각적 작업 도구 및 편집 인터페이스 강화 |
| `feat(desktop): 결제/위험 도구 실행 확인 다이얼로그` | 에이전트의 오작동 방지를 위한 사용자 승인 단계 추가 |
| `refactor(discord-bot): CLI 호출을 tory_llm::cli 로 위임` | 봇 엔진과 코어 로직 간의 인터페이스 단일화 |
| `chore(cli): 사용하지 않는 import/파라미터 제거` | 코드 클린업 및 컴파일 최적화 |

## 💡 작업 하이라이트

**실행 엔진(Execution Engine)의 도메인 확장**

이번 작업의 핵심은 Tory가 단순히 대화하는 수준을 넘어 실제 하드웨어 제어와 외부 API 기반의 복잡한 업무를 수행하도록 만든 것입니다. 특히 영상 편집 파이프라인의 구축은 Whisper STT와 영상 분할 로직을 결합하여, 에이전트가 멀티미디어 자산을 직접 가공할 수 있는 능력을 갖추게 했습니다. 이는 Tory의 아키텍처 중 실행 엔진이 도구(Tool)를 선택하고 실행하는 과정이 더욱 정교해졌음을 보여줍니다.

**보안 및 비용 최적화를 위한 로컬 가이드라인**

클라우드 기반 LLM은 높은 성능을 보이지만 비용과 개인정보 유출의 우려가 있습니다. 이를 해결하기 위해 사용자의 하드웨어 성능을 분석하여 로컬 모델 실행 가능 여부를 판단하는 기능을 도입했습니다. 시스템 사양에 맞는 최적의 모델(예: 8B 이하 모델)을 추천함으로써, 사용자가 별도의 설정 지식 없이도 개인화된 보안 환경을 구축할 수 있도록 지원합니다. 이는 Tory가 지향하는 '자율 비서'로서의 사용자 배려 측면을 강화한 것입니다.

**안전한 자율성을 위한 Human-in-the-loop 설계**

자율 AI 시스템에서 발생할 수 있는 잠재적 위험을 관리하기 위해 '위험 도구 실행 확인' 다이얼로그를 구현했습니다. 결제 API 호출이나 중요 데이터 접근 등 시스템이 정의한 정책(Policy)에 위배되거나 민감한 동작이 감지될 경우, 오케스트레이터는 실행 엔진을 즉시 가동하지 않고 사용자에게 최종 승인을 요청합니다. 이는 에이전트의 자율성과 사용자의 통제권 사이의 균형을 맞추는 핵심적인 안전장치입니다.
