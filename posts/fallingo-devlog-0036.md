# fallingo 개발일지 - 59fedb2..9995601 (35개 커밋)

안녕하세요! Fallingo 개발자 Su입니다. 👨‍💻
어느덧 베타 런칭 목표 시점인 2025년 12월이 정말 코앞으로 다가왔네요. 이번 개발일지는 런칭을 앞두고 막바지 스퍼트를 올린 35개의 커밋 내역을 바탕으로 작성했어요. 짧은 기간 동안 정말 많은 기능들이 추가되고 안정화 작업이 이루어졌습니다. 그럼 바로 시작해볼까요? 😊

**작업 기간**: 2025-11-18

## 📝 이번 기간 작업 내용

이번 기간에는 AI 기반 추천 시스템 고도화부터 검색 기능 최적화, 그리고 API 버저닝 작업까지, Fallingo의 핵심 기능들을 탄탄하게 다지는 데 집중했습니다. 🏃‍♂️

### 1. AI/추천 시스템 및 게이미피케이션 완성 ✨
*   **AI 추천 시스템 고도화 및 포인트 시스템 완성**: Fallingo의 핵심인 AI 추천 시스템을 더욱 정교하게 만들고, 사용자 참여를 유도하는 포인트 시스템을 드디어 완성했어요! 🥳 맛집 방문 시 OCR 영수증 인증으로 보너스 포인트를 지급하고, 북마크한 장소를 방문하면 추가 점수를 주는 방식으로 재미를 더했습니다.
*   **대장금 티어 시스템 구현**: 사용자들의 활동 레벨을 직관적으로 보여주는 '대장금 티어' 시스템도 구현했어요. ⭐~⭐⭐⭐ 별 개수로 티어를 표시하고, 특정 티어(Tier 3+) 이상만 접근할 수 있는 프로필 제한 기능도 추가해서 상위 티어에 대한 동기 부여를 주었습니다.

```
- feat: AI 추천 시스템 고도화 및 포인트 시스템 완성
- feat: 북마크 장소 방문 OCR 보너스 +7점 구현
- feat: Tier 3+ 대장금 프로필 접근 제한 구현
- feat: 대장금 티어별 별 개수 표시 (⭐~⭐⭐⭐)
```
이 섹션에서 총 4개의 핵심 `feat` 기능이 완성되었습니다.

### 2. 검색 기능 및 성능 최적화 🔍
*   **PostgreSQL Full-text search 구현**: 사용자 경험에 가장 중요한 검색 성능을 위해 PostgreSQL의 Full-text search를 도입하고 성능 최적화 작업을 진행했습니다. 덕분에 이제 원하는 맛집 정보를 훨씬 빠르고 정확하게 찾을 수 있게 되었어요. 🚀
*   **검색 자동완성 및 최근 검색어**: 사용자 편의성을 높이기 위해 검색 자동완성 기능과 최근 검색어 로컬 저장 기능을 구현했습니다. 📱
*   **API v2 검색 특화**: 기존 검색 API를 `v2`로 업데이트하며, `type` 파라미터를 제거하고 추천 맛집 특화 검색을 제공하도록 개선했습니다.

```
- feat: PostgreSQL Full-text search 구현 (성능 최적화)
- feat: 검색 자동완성 및 최근 검색어 로컬 저장 구현
- feat: 검색 API v2 - type 파라미터 제거 및 추천맛집 특화 검색
- feat: 추천맛집 검색 필터 구현
```
검색 기능 관련 4개의 `feat`와 1개의 `docs` (인덱스 확인) 작업이 진행되었습니다.

### 3. API 버저닝 및 시스템 안정화 🏷️
*   **API v1 버저닝 통일**: 모든 API 호출 경로에 `/v1/` prefix를 추가하여 버저닝을 통일하는 작업을 진행했습니다. 프론트엔드에서도 이에 맞춰 API 경로를 업데이트하고 연도 필터 위젯 등을 추가했어요. 이는 향후 시스템 확장성과 유지보수성을 크게 높여줄 중요한 작업이었습니다.

```
- feat: 모든 API 호출에 /v1/ 버전 prefix 추가
- feat: 프론트엔드 API v1 경로 업데이트 및 연도 필터 위젯 추가
- feat: API v1 버저닝 통일 및 추천맛집 검색 필터 구현
```
3개의 `feat` 커밋으로 API 버저닝을 통일했습니다.

### 4. 주요 UI/UX 기능 구현 및 개선 🎨
*   **피드 상세, 공유, 댓글 기능**: 피드 상세 화면과 공유 기능을 구현하고, 댓글 화면과 API 연동까지 마쳤습니다. 사용자 간 소통을 위한 핵심 기능들이죠! 💬
*   **피드 좋아요/북마크 (낙관적 업데이트)**: 피드 좋아요 및 북마크 기능을 API와 연동하면서, 사용자 경험을 위해 낙관적 업데이트(Optimistic UI Update)를 적용했어요. 즉시 UI에 반영되어 사용자가 빠르게 피드백을 받을 수 있습니다.
*   **메인 화면 작성 플로우 간소화**: 메인 화면에서 플로팅 버튼을 제거하여 피드 작성 플로우를 더 직관적이고 간소하게 변경했습니다.

```
- feat: 피드 상세 화면 및 공유 기능 구현
- feat: 댓글 화면 API 연동 완료
- feat: 피드 좋아요/북마크 API 연동 (낙관적 업데이트)
- refactor: 메인 화면 작성 플로우 간소화 (플로팅 버튼 제거)
```
총 5개의 `feat`와 1개의 `refactor`로 사용자 경험을 개선했습니다.

### 5. 클라우드 서비스 연동 및 인증 강화 ☁️
*   **GCP Identity Platform & Twitter OAuth**: GCP Identity Platform을 통한 인증 코드 검증을 완료하고, Twitter OAuth 백엔드 플로우를 구현하여 API Key 노출을 방지했습니다. 보안을 위해 WebView 대신 브라우저 리다이렉트를 사용하도록 변경했어요.
*   **Vision API & GCS**: OCR 영수증 인증과 피드 등록 시 Vision API를 활용하고, 이미지 업로드를 위한 GCS (Google Cloud Storage) 구현도 완료했습니다. 덕분에 영수증 검증과 이미지 관리가 더욱 효율적이고 안전해졌습니다.

```
- docs: GCP Identity Platform 코드 검증 완료
- refactor: Twitter OAuth를 WebView 대신 브라우저 리다이렉트로 변경
- feat: Twitter OAuth 백엔드 플로우 구현 (API Key 노출 방지)
- docs: Vision API 및 피드 등록 검증 완료 표시 (2회)
- docs: GCS 이미지 업로드 구현 완료 표시
```
이 분야에서 3개의 `feat`, 1개의 `refactor`, 4개의 `docs` 커밋이 있었습니다.

### 6. 데이터베이스 및 성능 최적화 💾
*   **데이터베이스 인덱스 90개 적용**: 프로젝트 전반에 걸쳐 데이터베이스 인덱스를 90개 적용하여 쿼리 성능을 대폭 향상시켰습니다. 📈 빠른 데이터 처리는 Fallingo의 핵심 경쟁력이 될 거예요.
*   **Cloud SQL 법률 문서 시딩**: 국가 선택 및 법률 문서 API 연동과 함께 Cloud SQL에 법률 문서를 시딩하는 작업도 완료했습니다.

```
- docs: 데이터베이스 인덱스 90개 적용 확인 (2025-11-18)
- feat: 국가 선택 및 법률 문서 API 연동 구현
- docs: Cloud SQL 법률 문서 시딩 완료 확인
```
데이터베이스 관련 1개의 `feat`와 2개의 `docs` 커밋이 있었네요.

### 7. 기타 개선 및 문서화 📑
*   **AdMob 통합**: 수익 모델을 위한 AdMob 통합을 완료하고 백엔드 검증까지 마쳤습니다.
*   **체크리스트 업데이트**: 개발 진행 상황을 투명하게 확인하기 위한 `docs`성 커밋이 많았습니다. 특히 체크리스트 항목을 `[x]`에서 `✅`로 변경하고, 날짜도 정확히 수정하는 등 꼼꼼하게 관리했어요.
*   **피드 카드 개선**: 피드 카드에서 인증 배지를 제거하고, 이미지와 지도를 토글할 수 있는 기능도 확인했습니다.

```
- feat: AdMob 통합 구현 및 백엔드 검증 완료
- docs: 체크리스트 [x] → ✅ 표시로 변경 (다수)
- refactor: 피드 카드 인증 배지 제거
- docs: 피드 카드 지도 토글 기능 완료 확인 (이미지↔지도 전환)
```
1개의 `feat`, 1개의 `refactor`와 다수의 `docs` 커밋으로 전반적인 프로젝트 관리를 했습니다.

## 💡 작업 하이라이트

이번 기간의 가장 의미 있는 하이라이트는 역시 **AI 추천 시스템의 완성**과 **PostgreSQL Full-text search 도입으로 인한 검색 성능의 비약적인 향상**이라고 생각합니다.

1.  **AI 추천 시스템 및 포인트 시스템 완성**: Fallingo의 정체성을 만드는 핵심 기능이라고 할 수 있습니다. 사용자가 단순히 정보를 얻는 것을 넘어, 즐거움을 느끼고 계속해서 플랫폼에 참여하도록 유도하는 중요한 시스템이에요. 베타 런칭 전에 이 부분을 단단하게 다질 수 있어서 정말 뿌듯합니다.
2.  **PostgreSQL Full-text search 및 90개 인덱스 적용**: 사용자가 많아질수록 가장 먼저 체감하는 부분이 바로 검색 속도와 정확도일 텐데요. PostgreSQL의 강력한 Full-text search 기능을 활용하고, 데이터베이스 전체에 90개에 달하는 인덱스를 적용하면서 성능 최적화에 많은 공을 들였습니다. 이 덕분에 Fallingo는 방대한 맛집 데이터 속에서도 사용자에게 빠르고 정확한 정보를 제공할 수 있게 되었어요. 📈

그리고 **API v1 버저닝 통일**은 당장의 사용자에게 체감되는 부분은 아닐 수 있지만, 장기적으로 Fallingo가 성장하고 확장하는 데 있어서 매우 중요한 기반을 마련한 작업이었습니다. 개발 관점에서 시스템의 안정성과 유지보수성을 크게 높여줄 것이라고 기대하고 있습니다.

반복적으로 `docs` 커밋이 많았던 이유는 베타 런칭이 정말 코앞으로 다가오면서, 기능 개발만큼이나 **"현재 상태를 정확히 파악하고 공유하는 것"**이 중요해졌기 때문입니다. 체크리스트를 꼼꼼히 업데이트하고, 각 기능별 검증 완료 표시를 확실히 하면서 혹시 빠진 부분은 없는지 점검하는 데 많은 시간을 할애했습니다.

남은 기간 동안은 주로 다음과 같은 작업에 집중할 예정입니다.

*   성능 최적화 및 안정성 테스트 (특히 부하 테스트)
*   사용자 피드백을 반영한 마지막 디테일 조정
*   운영 환경 배포 준비 및 모니터링 시스템 구축

정말 긴 여정이었는데, 이렇게 목표했던 베타 런칭 시점에 맞춰 Fallingo의 모습을 갖춰나가고 있어서 감격스럽습니다. Google for Startups Cloud Program의 지원 덕분에 클라우드 인프라 걱정 없이 개발에만 집중할 수 있었던 것도 큰 도움이 되었습니다. 💖

다음 개발일지에서는 아마 베타 런칭 후의 소식과 사용자 피드백을 공유할 수 있기를 기대해봅니다!
항상 Fallingo에 관심 가져주시고 응원해주시는 모든 분들께 감사드립니다! 🙏
