# tory 개발일지 #29 - McEngine 최적화 + GPT-5.4 모델 확장 (10개 커밋)

**작업 기간**: 2026-03-19 ~ 2026-03-20

## 📝 이번 기간 작업 내용

### McEngine 개선 + UI 정비 (5개 커밋)

McEngine 출시 직후 성능과 UX 두 방향에서 개선을 진행했다. 메시지 분리 문제와 updateMessage 성능 이슈를 수정했다. MC 진행 단계별 안내 문구를 "생각 중", "질문", "종합" 단계에 맞게 구체화했다. 회의 이벤트 핸들러를 재설계해 메시지 분리와 디버그 로그를 정비했다. 참여자 제한을 기존 4명에서 해제해 전체 활성 프로필을 MC에 제공하도록 변경했다.

| 커밋 | 내용 |
|------|------|
| `fix(desktop): 회의 진행 상태 표시 + updateMessage 성능 개선` | 성능 개선 |
| `fix(meeting): MC 진행 상태 표시 개선 — 생각 중/질문/종합 단계별 안내` | 진행 안내 개선 |

| `feat(secretary): 회의 참여자 4명 제한 제거 — 전체 활성 프로필을 MC에 제공` | 참여자 제한 해제 |
| `fix: biome lint 수정 — import 정렬 + 미사용 private member 제거` | 린트 정리 |

### McEngine 계획→실행 분리 최적화 (1개 커밋)

McEngine의 가장 큰 병목이던 head LLM 호출 횟수를 N회에서 2회로 줄였다. 기존 구조는 계획 단계와 실행 단계가 뒤섞여 있어 각 질문마다 head가 별도로 개입했다. 계획과 실행을 명확히 분리해 계획 수립 1회, 실행 후 종합 1회로 축소했다. 회의 진행 시간이 실질적으로 단축됐다.

| 커밋 | 내용 |
|------|------|
| `perf(meeting): McEngine 계획→실행 분리 — head LLM 호출 N회→2회로 감소` | LLM 호출 최적화 |

### OAuth 재인증 버그 수정 (1개 커밋)

Token Rotation이 미반영돼 매번 OAuth 재인증이 요구되는 버그를 수정했다. 세션이 SQLite에 영속화됐지만 토큰 갱신이 제대로 반영되지 않아 앱을 재시작할 때마다 로그인이 필요했다. 진행 상태 UI도 함께 개선해 배너가 올바르게 활성화되고 불필요한 시스템 메시지가 제거됐다.

| 커밋 | 내용 |
|------|------|
| `fix(desktop): 회의 진행 상태 UI 개선 — 배너 활성화 + 시스템 메시지 제거` | 진행 상태 UI 개선 |
| `fix(desktop): Token Rotation 미반영으로 매번 OAuth 재인증되는 버그 수정` | OAuth 버그 수정 |

### 모델 확장 + 서버 카탈로그 동기화 (2개 커밋)

모델 tier 구분을 제거하고 GPT-5.4 Mini와 GPT-5.4 Nano를 모델 카탈로그에 추가했다. 서버 모델 카탈로그를 데스크톱 앱이 직접 동기화하고 자동 활성화하는 기능도 구현했다. 서버에서 새 모델이 추가되면 클라이언트에서 별도 업데이트 없이 반영된다.

| 커밋 | 내용 |
|------|------|
| `feat(shared): 모델 tier 구분 제거 + GPT-5.4 Mini/Nano 추가` | 모델 카탈로그 확장 |
| `feat(desktop,server): 서버 모델 카탈로그 직접 동기화 + 자동 활성화` | 카탈로그 동기화 |

## 💡 작업 하이라이트

**N회 호출에서 2회로**

McEngine은 회의를 구조화하기 위해 head LLM이 여러 차례 개입한다. 초기 구현은 각 참여자 발언마다 head가 방향을 재검토했다. 이는 품질보다 지연을 더 많이 만들어냈다. 계획 단계에서 전체 방향을 한 번 수립하고 실행 후 종합만 한 번 더 수행하는 방식으로 개편했다. 회의 품질은 유지되면서 속도가 크게 향상됐다.

**서버 카탈로그 동기화**

모델 카탈로그를 서버에서 직접 동기화하면 새 모델 지원을 앱 업데이트 없이 배포할 수 있다. 서버가 진실의 단일 출처가 되고 클라이언트는 그것을 따라가는 구조다. GPT-5.4 Mini/Nano 추가도 이 방향성의 시작이다.
