# vibe 개발일지 #52 - v3.2.0 릴리즈와 아키텍처 대청소 (10개 커밋)

**작업 기간**: 2026-07-02 ~ 2026-07-08

안녕하세요, vibe 패키지를 개발하고 있는 Su입니다. 

"바이브 코딩(vibe coding)은 편하게, 하지만 최소한의 퀄리티는 보장되게(Easy vibe coding + Minimum quality guaranteed)"라는 철학으로 시작한 vibe 프로젝트가 드디어 **v3.2.0**을 맞이했습니다. 

이번 일주일은 그동안 빠르게 달리느라 쌓였던 코드 부채를 청산하는 '대청소' 기간이었습니다. AI가 코드를 마음껏 작성하더라도 뼈대가 흔들리지 않도록 하네스(Harness)를 더 단단하게 조이는 작업들이 주를 이루었는데요. 어떤 고민과 변화가 있었는지 공유해 드립니다.

---

## 📝 이번 기간 작업 내용

이번 10개의 커밋은 주로 시스템의 안정성을 높이고, 레거시 구조를 분해하여 유연성을 확보하는 데 집중되었습니다.

### 1. 코어 엔진 리팩토링 & 코드 다이어트 (약 -1,500줄 최적화)
바이브 코딩을 하다 보면 AI가 중복된 유틸리티나 불필요한 훅을 만들어내곤 합니다. 이를 주기적으로 청소해주지 않으면 엔진이 무거워집니다.
* **`vibe.utils` 해체 및 스킬 문서화 분리**: 거대했던 `vibe.utils`를 해체하고, 지나치게 무거웠던 `SKILL.md`를 `references/` 디렉토리로 쪼개어 관리하기 시작했습니다. AI 에이전트가 컨텍스트를 읽을 때 필요한 부분만 효율적으로 참조할 수 있도록 구조화했습니다.
* **CLI 설치 배선 및 훅 최적화**: 중복되거나 쓰이지 않는 죽은 코드(Dead code)와 훅을 대거 제거하고, CLI의 설치 프로세스를 깔끔하게 단일 파이프라인으로 통합했습니다.

### 2. 하네스 안정성 강화를 위한 정적 테스트 도입
* **참조 무결성(Reference Integrity) 테스트 신설**: 에이전트, 스킬, 훅이 서로를 참조할 때 깨진 링크가 없는지 검증하는 정적 테스트를 추가했습니다. AI가 코드 생성 중 링크를 실수로 깨뜨려도 이 테스트 덕분에 CI 단계에서 즉시 차단됩니다.
* **CI 빌드 스텝 보완**: `dist`에 의존하는 훅 테스트들이 CI 빌드 환경에서도 안정적으로 통과할 수 있도록 `tsc` 빌드 과정을 테스트 잡(Job)에 명시적으로 추가했습니다.

### 3. 보안 인프라 및 오픈소스 거버넌스 개선
* **NPM 배포 OIDC 전환**: 정적 토큰 방식에서 탈피하여 GitHub Actions와 NPM 간의 OIDC(OpenID Connect)를 통한 **Trusted Publishing** 설정을 최종 완료했습니다. 이제 더 안전하고 투명한 자동 배포가 가능합니다.
* **스타 게이트(Star Gate) 도입**: 오픈소스 메인테이너로서의 최소한의 방어선입니다. 리포지토리에 스타(Star)를 누르지 않고 무분별한 이슈나 PR을 생성하는 경우, 자동으로 안내 메시지를 띄우고 닫는 GitHub Actions 워크플로우를 구축했습니다.

## 💡 작업 하이라이트

### 1. "Agent = Model + Harness" 철학을 위한 '참조 무결성 정적 테스트'
vibe는 단순한 프롬프트 도구가 아닙니다. AI 모델이 안전하게 뛰어놀 수 있는 **하네스(Harness)** 역할을 해야 합니다. 

바이브 코딩을 하다 보면 AI가 새로운 스킬을 만들면서 기존 훅의 참조 경로를 바꾸거나, 템플릿 파일명을 실수로 다르게 적는 일이 종종 발생합니다. 런타임에 에러를 마주하면 이미 늦습니다.

이를 해결하기 위해 **설치·훅·에이전트 간의 연결 관계를 정적으로 전수 조사하는 테스트 코드**를 신설했습니다.

```typescript
// test/integrity.test.ts (의사 코드)
test("모든 Agent가 참조하는 Hook과 Skill은 실제로 존재해야 한다", () => {
  const agents = loadAllAgents();
  const hooks = loadAllHooks();
  
  agents.forEach(agent => {
    agent.requiredHooks.forEach(hookId => {
      expect(hooks).toContain(hookId); // 참조 깨짐 방지!
    });
  });
});
```
이 테스트 도입 이후, AI 에이전트가 코드를 개선하는 과정에서 발생할 수 있는 '잠재적 먹통' 현상을 사전 차단할 수 있게 되었습니다. 진정한 **"Minimum quality guaranteed"**의 실현입니다.

### 2. `--sub` 메뉴 탐색 기능 추가 (`feat(clone)`)
템플릿이나 보일러플레이트를 복제할 때 전체를 가져오는 대신, 하위의 서브 메뉴(Sub-menu)들을 인터랙티브하게 탐색하고 선택적으로 클론할 수 있는 기능을 추가했습니다. 이제 사용자는 더욱 가볍고 필요한 부분만 쏙쏙 골라 vibe 워크플로우에 통합할 수 있습니다.
