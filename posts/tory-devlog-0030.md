# tory 개발일지 #30 - 자율 실행 + 행사 자동화 + 개인 기억 시스템 (10개 커밋)

**작업 기간**: 2026-03-21 ~ 2026-03-24

## 📝 이번 기간 작업 내용

### 자율 실행 루프 안전화 + 브라우저 도구 (1개 커밋)

`closed-loop` 크레이트에 승인 게이트와 파일 잠금을 추가했다. 자율 실행 루프가 파일 시스템이나 외부 서비스에 접근할 때 사용자 승인을 거치는 구조다. 파일 잠금은 여러 에이전트가 동시에 같은 파일을 수정하는 경우를 방지한다. 도구 실행 통합도 이 커밋에서 이뤄졌으며 브라우저 도구도 함께 추가했다.

| 커밋 | 내용 |
|------|------|
| `feat(tools,closed-loop): 승인 게이트 + 파일 잠금 + 도구 실행 통합 + 브라우저 Tool` | 자율 실행 안전화 |

### 컨텍스트 윈도잉 + 회의 토큰 예산 (1개 커밋)

회의 진행 중 컨텍스트가 과도하게 쌓이는 문제를 해결하기 위해 컨텍스트 윈도잉을 구현했다. 모델 비용 계층을 정의하고 회의별 토큰 예산을 설정했다. 저비용 모델을 우선 활용하고 예산이 소진되면 발언 횟수를 조정하는 방식으로 비용을 통제한다.

| 커밋 | 내용 |
|------|------|
| `feat(meeting,shared): 컨텍스트 윈도잉 + 모델 비용 계층 + 회의 토큰 예산` | 토큰 예산 관리 |

## 💡 작업 하이라이트

**자율 실행과 안전의 균형**

자율 에이전트가 파일 시스템에 접근하거나 외부 서비스를 호출할 때 제어 없이 작동하면 위험하다. 승인 게이트는 이 균형을 위한 장치다. 모든 작업에 승인이 필요한 것은 아니지만, 파일 수정이나 외부 호출처럼 되돌리기 어려운 작업에는 반드시 사람의 확인을 거친다. 파일 잠금은 멀티 에이전트 환경에서 충돌을 방지하는 인프라다.

**컴파운딩 기억**

단순한 대화 이력 저장과 개인 컴파운딩 시스템은 다르다. 컴파운딩은 시간이 지남에 따라 기억이 누적되고 더 유용해지는 구조를 목표로 한다. 토리가 오래 사용될수록 주인의 취향, 작업 스타일, 자주 쓰는 패턴을 더 잘 이해하게 된다. 이 방향으로 꾸준히 발전시켜 나간다.
