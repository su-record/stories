# vibe 개발일지 #35 - 글로벌 스탠다드 확립과 하네스 엔지니어링의 진화 (27개 커밋)

**작업 기간**: 2026-04-09 ~ 2026-04-10

안녕하세요, Su입니다. 
최근 `vibe` 프로젝트를 진행하며 'AI 코딩에서 하네스(Harness)의 역할은 어디까지인가'에 대해 깊은 고민이 있었습니다. 에이전트는 단순히 모델의 성능에 의존하는 것이 아니라, 견고한 가이드라인과 품질 보장 체계(Harness)가 결합될 때 비로소 완성된다는 제 철학을 바탕으로, 이번 27개의 커밋에는 그 고민의 결과물들을 꾹꾹 눌러 담았습니다.

## 📝 이번 기간 작업 내용

### 1. 전면적인 글로벌 스탠다드 적용 (영문화 및 네이밍 정리)
vibe가 더 넓은 생태계에서 쓰일 수 있도록 내부 코어 스킬들을 영문화하고 직관적으로 재정의했습니다.
- **전체 44개 스킬 영문 통일**: `vibe.plan`, `vibe.spec`, `vibe.figma` 등 모든 스킬의 인터페이스와 응답 메시지를 영문으로 재작성했습니다.
- **Discover → Interview 변경**: '발견'이라는 추상적인 개념보다 사용자에게 요구사항을 묻고 답하는 '인터뷰'가 더 적합하다고 판단하여 `vibe.interview`로 스킬명을 변경했습니다.
- **커맨드 일원화**: `/vibe.spec.review` 같은 파편화된 커맨드를 제거하고 스킬 시스템 내로 통합하여 워크플로우를 단순화했습니다.

### 2. 하네스 엔지니어링(Harness Engineering) 6축 도입
단순한 코드 생성을 넘어 프로젝트를 분석하고, 비계를 설정하며, 진단하는 체계를 구체화했습니다.
- **6축 기반 개선**: `analyze`, `scaffold`, `harness` 진단 기능을 확장했습니다.
- **CLAUDE.md 자동 생성**: 프로젝트 분석 결과를 바탕으로 Claude Code가 참조할 수 있는 프로젝트 컨텍스트 파일을 자동으로 생성하는 기능을 추가했습니다.
- **프로젝트 타입 감지**: 라이브러리나 패키지 형태의 프로젝트일 경우 불필요한 문서화 항목을 N/A 처리하는 등 지능형 감지 로직을 추가했습니다.

### 3. 품질 게이트(Quality Gate)의 고도화
무조건 '차단'하는 것이 능사가 아님을 깨닫고, 더 유연한 흐름을 만들었습니다.
- **Minimum Viable Plan (MVP) Gate**: 인터뷰 커버리지가 50% 미만일 경우 플랜 생성을 차단하여, 불충분한 정보로 코드가 짜여지는 것을 방지합니다.
- **Gap Filling 시스템**: 기존의 단순 차단(Blocking) 게이트를 'AI-Driven Gap Filling'으로 교체했습니다. 부족한 부분이 있다면 AI가 스스로 메우거나 사용자에게 제안합니다.
- **리뷰 루프 최적화**: 라운드 제한(Cap)을 제거하고, 대신 '100점 수렴 기반 종료' 방식을 도입하여 품질이 확보될 때까지 집요하게 검토합니다.

### 4. Figma 파이프라인 및 인터랙션 강화
- **3-branch Router**: Figma 관련 로직을 READ/WRITE/Idempotent 분기로 재구성하여 상태 관리의 안정성을 높였습니다.
- **Stuck → Ask User 패턴**: AI가 루프에 빠지거나 판단이 모호한 상황(Figma, Verify, Run 과정 등)에서 사용자에게 즉시 질문하고 에스컬레이션할 수 있는 경로를 구축했습니다.

## 💡 작업 하이라이트

### "차단하기보다, 스스로 채우기" (Gap Filling)
이전 버전에서는 요구사항이 부족하면 에이전트가 "더 알려주세요"라고 하며 멈춰버리는 경우가 많았습니다. 하지만 이번 업데이트에서는 **AI-Driven Gap Filling**을 도입했습니다. 하네스가 판단하기에 치명적인 결함이 아니라면, AI가 기존 컨텍스트를 바탕으로 논리적인 공백을 메운 뒤 사용자에게 확인을 받는 방식으로 전환했습니다. 덕분에 `/vibe.spec` → `/vibe.run`으로 이어지는 흐름이 훨씬 매끄러워졌습니다.

### "Harness Engineering 6축"
README를 리라이트하며 명문화한 '하네스 엔지니어링 6축'은 vibe의 핵심 정체성입니다. 
1. **Analyze**: 맥락 파악
2. **Scaffold**: 구조 설계
3. **Harness**: 안전장치 마련
4. **Interview**: 의도 확인
5. **Plan**: 전략 수립
6. **Verify**: 검증
이 6개 축이 유기적으로 돌아가며 "Easy vibe coding + Minimum quality guaranteed"라는 약속을 지킵니다.
