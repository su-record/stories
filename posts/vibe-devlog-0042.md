# vibe 개발일지 #42 - SSOT 강화와 Claude Code 정식 지원 (10개 커밋)

**작업 기간**: 2026-04-17

안녕하세요, Su입니다. 

오늘 하루는 정말 숨 가쁘게 지나갔네요. 2.9.28 버전부터 2.9.31 버전까지 총 4번의 릴리즈가 있었습니다. "Agent = Model + Harness"라는 제 철학을 완성하기 위해, 오늘은 특히 하네스(Harness)의 안정성과 사용자 경험을 개선하는 데 집중했습니다.

단순히 AI가 코드를 짜게 내버려 두는 것이 아니라, 그 결과물이 우리가 의도한 퀄리티를 유지할 수 있도록 돕는 시스템을 다듬는 과정이었습니다. 오늘 작업한 10개의 커밋을 중심으로 주요 변경 사항을 공유해 드립니다.

## 📝 이번 기간 작업 내용

이번 작업은 크게 **프로젝트 구조 최적화**, **Claude Code(coco) 연동 강화**, **Figma 파이프라인 정교화** 세 가지 축으로 진행되었습니다.

### 1. 프로젝트 자산의 SSOT(Single Source of Truth) 확립
AI 도구가 프로젝트를 이해할 때 가장 중요한 것은 "어디를 봐야 하는가"를 명확히 하는 것입니다.
*   **SSOT 통합**: 프로젝트 자산들을 `.vibe/` 디렉토리 아래로 일원화했습니다. 흩어져 있던 설정과 리소스들을 한곳에 모음으로써 AI가 맥락을 파악하는 효율을 극대화했습니다.
*   **마커 중복 제거**: `VIBE:END` 마커가 중복 생성되던 버그를 수정하여 파일 구조의 깔끔함을 더했습니다.

### 2. Claude Code (coco) 및 CLI 생태계 대응
최근 핫한 Claude Code와의 협업 능력을 끌어올렸습니다.
*   **coco 최적화**: README와 `setup.sh`를 Claude Code 환경에 맞게 조정했습니다. 이제 `coco` 환경에서도 `vibe`가 물 흐르듯 작동합니다.
*   **상태 노출(Status Exposure)**: `feat(status)`를 통해 현재 사용 중인 벤더(Vendor), 가용성, 메서드 등을 한눈에 확인할 수 있게 하여 디버깅 편의성을 높였습니다.

### 3. Figma to Code 퀄리티 가드 (Harness의 핵심)
vibe의 존재 이유인 '최소 품질 보장'을 위한 기능입니다.
*   **CSS 손실 탐지**: Figma 디자인을 코드로 옮길 때 발생하는 미세한 속성 누락을 잡기 위해 `raw` 데이터와 `audit` 결과를 대조하는 로직을 추가했습니다.
*   **유연한 파일 생성 규칙**: "새 파일을 함부로 만들지 말 것"이라는 엄격한 규칙이 때로는 개발자의 명시적인 요청까지 막아버리더군요. 사용자가 직접 요청할 때는 파일을 생성할 수 있도록 규칙을 살짝 완화했습니다.

## 💡 작업 하이라이트

### 🏠 모든 길은 `.vibe/`로 통한다 (SSOT)
기존에는 설정 파일들이 루트에 여기저기 흩어져 있었습니다. 이는 개발자에게도 지저분해 보일 뿐만 아니라, AI 에이전트가 "무엇이 중요한 파일인지" 판단하는 데 노이즈가 됩니다. 이번 **`.vibe/` SSOT 리팩토링**을 통해 에이전트에게 명확한 '홈 디렉토리'를 제공하게 되었습니다.

### 🎨 Figma 번역기, 감시자를 두다
AI가 Figma 디자인을 CSS로 변환할 때, 가끔 복잡한 그라데이션이나 그림자 효과를 슬쩍 빼먹는 경우가 있습니다. 이번에 추가된 **Audit 시스템**은 원본 디자인 데이터와 생성된 결과물을 대조합니다. 만약 중요한 속성이 누락되었다면, 하네스가 이를 감지하고 AI에게 재작업을 요청합니다. 이것이 바로 제가 지향하는 "Minimum quality guaranteed"입니다.

## 📊 개발 현황

- **버전 변화**: `v2.9.28` ⮕ `v2.9.31`
- **주요 수치**:
  - 프로젝트 설정 파일 위치 100% 단일화 (`.vibe/`)
  - Claude Code(coco) 호환성 패치 완료
  - Figma 속성 누락 감지 로직 탑재
