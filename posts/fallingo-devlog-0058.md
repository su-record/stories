### 1. CI/CD 및 iOS 빌드 환경 안정화 (5 Commit) 🛠️

Flutter를 사용하면서 가장 까다로운 부분은 역시 플랫폼별 환경 설정입니다. 특히 CI/CD 환경에서 iOS 빌드는 변수가 많아 주기적으로 손봐줘야 하는데요. 이번에는 몇 가지 근본적인 문제들을 해결했습니다.

| 커밋 내용 | 유형 | 상세 설명 |
| :--- | :--- | :--- |
| `fix: CI 스크립트에서 Pods 캐시 정리 추가` | CI/CD | CI 과정 중 이전 빌드의 캐시 잔여물로 인한 오작동 방지. `Pods` 캐시를 명시적으로 정리했습니다. |
| `fix: iOS deployment target 12.0 → 13.0으로 상향` | 빌드 설정 | 최신 Flutter 및 패키지들과의 호환성을 높이고, 레거시 환경 지원 부담을 줄이고자 최소 배포 버전을 상향했습니다. |

## 💡 작업 하이라이트

### 1. 뼈 아팠던 인증 500 에러 해결 (Critical Fix)

가장 시간을 많이 잡아먹었던 부분은 회원가입 직후 로그인 시 발생하는 **500 에러**였습니다. 이 버그는 사용자가 서비스를 사용하지 못하게 만드는 블로커(Blocker)였기 때문에 우선순위가 높았습니다.

FastAPI 백엔드에서 회원 정보가 생성된 후, 즉시 인증 토큰을 생성하는 과정에서 데이터베이스 트랜잭션 커밋 순서에 미묘한 문제가 있었습니다. AI 기반으로 코드를 작성하다 보니 놓친 부분이 있었고, 결국 DB 쿼리 로그를 직접 분석하며 문제를 해결할 수 있었습니다.

이 경험을 통해, AI가 코드를 생성해주더라도 **인증/결제와 같이 핵심 로직이 들어가는 부분은 트랜잭션 단위로 직접 검토**해야 한다는 점을 다시 한번 배웠습니다. 📝

### 2. 위치 기반 이벤트 시스템의 기반 마련 (New Feature)

`feat: 월드 이벤트 사전 노출 시스템 구현` 커밋을 통해 Fallingo 서비스의 재미 요소인 '월드 이벤트' 시스템의 뼈대를 구현했습니다. 이는 특정 시간, 특정 위치에서만 발동되는 한정판 미션을 사용자에게 사전에 노출하여 참여를 유도하는 기능입니다.

현재는 API 엔드포인트와 데이터 모델만 구현했지만, 다음 주에는 Flutter 프론트엔드에서 이 정보를 시각화하는 작업을 진행할 예정입니다. 이제 정말 '음식 소셜 플랫폼'다워지고 있다는 느낌을 받습니다.
