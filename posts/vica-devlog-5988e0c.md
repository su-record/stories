## vica 개발일지: CLI 모델 선택과 TUI 사용감 다듬기

이번 작업은 크게 보면 **“사용자가 지금 무엇을 쓰고 있는지 더 잘 보이게 만들기”**와 **“터미널 안에서의 대화 경험을 덜 불편하게 만들기”**에 집중했습니다. 기능을 새로 얹는 작업도 있었지만, 그보다 기존 흐름을 정리하고 손에 걸리는 부분을 줄이는 데 시간을 많이 썼습니다.

### 프로바이더 구조 정리와 CLI 실행자 모델 선택

먼저 프로바이더 관련 구조를 정리했습니다. vica는 여러 모델과 실행 방식을 다루다 보니, 내부적으로 “누가 어떤 모델을 실행하는가”가 조금씩 흐릿해질 수 있습니다. 이번에는 CLI 실행자 단위에서 모델을 선택할 수 있도록 흐름을 다듬었습니다.

사용자 입장에서는 단순히 모델을 고르는 기능처럼 보이지만, 내부적으로는 프로바이더와 실행자의 책임을 조금 더 분리하는 작업이었습니다. 앞으로 모델이 늘어나거나 실행 방식이 추가될 때도 덜 복잡하게 확장할 수 있도록 기반을 정리한 셈입니다.

### `/model` 메뉴 UX 개선

모델 선택 메뉴도 함께 손봤습니다. 기존 `/model` 화면은 기능은 동작하지만, 어떤 항목이 어떤 실행자에 속하는지 한눈에 보기에는 조금 아쉬웠습니다. 그래서 구분 라벨을 추가해 목록의 맥락이 더 잘 보이도록 했습니다.

또 하나 신경 쓴 부분은 선택 후 화면 갱신입니다. 모델을 선택했을 때 전체가 불필요하게 다시 그려지는 느낌을 줄이고, 필요한 sub 영역만 다시 고정되도록 조정했습니다. 이런 부분은 작은 차이지만 TUI에서는 꽤 크게 느껴집니다. 터미널 앱은 화면이 조금만 튀어도 사용자가 “방금 뭐가 바뀐 거지?” 하고 멈칫하게 되니까요.

### 입력 경험과 스킬 호출 사용성 개선

이번 업데이트에서 가장 체감이 큰 영역은 TUI 입력부입니다. 긴 텍스트를 붙여넣을 때 화면이 지저분해지는 문제를 줄이기 위해 **페이스트 접기**를 넣었고, 입력 중 긴 문장이 자연스럽게 이어지도록 **자동 줄바꿈**도 다듬었습니다.

또한 스킬을 조금 더 편하게 부를 수 있도록 **이름 기반 호출**을 보강했습니다. 명령어를 정확히 기억해야만 쓸 수 있는 도구는 점점 부담이 커지는데, 이름으로 자연스럽게 접근할 수 있으면 사용 빈도도 올라가고 진입 장벽도 낮아집니다.

여기에 현재 사용 중인 모델을 TUI에서 바로 확인할 수 있도록 표시도 추가했습니다. 모델을 바꿔가며 테스트하다 보면 “지금 내가 어떤 모델로 말하고 있지?”라는 순간이 자주 생기는데, 그 불확실성을 줄이고 싶었습니다.

### 웰컴 화면, 마크다운 렌더링, 라이브 통계

마지막으로 첫인상과 응답 표시 영역도 개선했습니다. 웰컴 마스코트를 더 전면에 배치해서 vica에 들어왔을 때의 느낌을 조금 더 살렸습니다. 기능만 있는 도구보다는, 계속 열어두고 싶은 도구가 되었으면 하는 마음이 있습니다.

응답 출력 쪽에는 터미널 마크다운 렌더링을 적용했습니다. 답변에 제목, 목록, 강조 같은 구조가 있을 때 훨씬 읽기 좋아졌습니다. 단순 텍스트로 쭉 밀려 나오는 것보다 정보의 결이 살아나서, 긴 답변을 볼 때 피로감이 줄어듭니다.

그리고 답변 생성 중에는 라이브 통계도 볼 수 있게 했습니다. 응답이 진행되는 동안 현재 상태를 더 잘 파악할 수 있어서, 기다리는 시간이 조금 덜 막막해졌습니다.

### 마무리

이번 4개 커밋은 화려한 단일 기능보다는, vica를 실제로 쓰면서 거슬렸던 부분들을 정리한 작업에 가깝습니다. 모델 선택은 더 명확하게, 입력은 더 편하게, 출력은 더 읽기 좋게 만드는 방향이었습니다.

아직 다듬을 부분은 많지만, 이번 작업을 통해 vica가 “기능이 되는 도구”에서 조금 더 “매일 쓰기 편한 도구” 쪽으로 가까워졌다고 느낍니다. 다음에는 이 기반 위에서 스킬 사용 흐름과 모델별 실행 경험을 더 자연스럽게 이어가 보고 싶습니다.
