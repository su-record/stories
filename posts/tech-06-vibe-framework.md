# Vibe: SPEC 주도 AI 코딩 프레임워크로 바이브 코딩을 더 쉽게

## 바이브 코딩, 더 체계적으로

AI 개발 방법론 시리즈에서 이야기했던 **"바이브 코딩을 위한 프레임워크"**가 완성되었습니다. 자연어 요구사항을 프로덕션 코드로 변환하는 체계적인 프레임워크, **Vibe**입니다.

## 왜 Vibe를 만들었나

fallingo를 개발하면서 수십 번의 리셋을 경험했습니다. 일관성이 무너지고, 문서와 코드가 어긋나고, 같은 실수를 반복했습니다.

순차적 문서 작성 방법론을 정립하고, SpecKit을 도입하면서 많이 개선되었지만, 여전히 불편한 점들이 있었습니다:

- **매번 처음부터 구조를 설계해야 함**
- **문서 작성 순서를 기억해야 함**
- **AI에게 같은 설명을 반복해야 함**
- **작업 단위를 수동으로 나눠야 함**

"이 모든 과정을 프레임워크로 만들면 어떨까?" 하는 생각에서 Vibe가 시작되었습니다.

---

## Vibe란?

**SPEC 주도 AI 코딩 프레임워크**입니다.

자연어 요구사항을 받아서:
1. EARS 기반 명세 작성
2. 기술 구현 계획 수립
3. 단계별 작업으로 분해
4. 실행 및 검증

이 모든 과정을 **체계적으로, 자동으로** 진행합니다.

---

## 핵심 특징

### 1. Claude Code 최적화

Claude Code에서 최적의 성능을 발휘하도록 설계되었습니다.

### 2. SPEC 주도 개발

**EARS (Easy Approach to Requirements Syntax)** 기반으로 명세를 작성합니다.

명확한 요구사항 정의로 일관성을 유지합니다.

### 3. 자동화된 계획 수립

기술 구현 계획을 자동으로 생성합니다:
- 아키텍처 설계
- 기술 스택 선정
- 비용 분석
- 위험 요소 식별

### 4. 작업 분해

기능을 단계별, 의존성을 고려한 작업으로 자동 분해합니다.

### 5. 38개 MCP 도구

hi-ai MCP의 38개 도구가 통합되어 있습니다:
- 코드 분석
- 품질 검사
- 성능 측정
- UI 미리보기
- 메모리 관리

### 6. 다국어 지원

한국어와 영어를 모두 지원합니다.

### 7. 7개 전문 에이전트

각 단계마다 전문화된 AI 에이전트가 작업합니다:
- **Specification Agent**: 명세 작성
- **Planning Agent**: 계획 수립
- **Task Agent**: 작업 분해
- **Backend Expert**: 백엔드 개발
- **Frontend Expert**: 프론트엔드 개발
- **Database Expert**: 데이터베이스 설계
- **Quality Reviewer**: 품질 검수

---

## 핵심 워크플로우

### 1. 프로젝트 초기화

```bash
vibe init
```

프로젝트 설정을 생성합니다.

### 2. 명세 작성

```bash
vibe spec "사용자 로그인 기능"
```

EARS 기반으로 기능 명세를 자동 생성합니다.

**생성되는 내용:**
- 기능 설명
- 요구사항 (필수/선택)
- 제약사항
- 예외 처리
- 검증 기준

### 3. 계획 수립

```bash
vibe plan "사용자 로그인 기능"
```

기술 구현 계획을 자동 생성합니다.

**생성되는 내용:**
- 아키텍처 설계
- 기술 스택 선정
- API 설계
- 데이터베이스 스키마
- 보안 고려사항
- 비용 추정

### 4. 작업 분해

```bash
vibe tasks "사용자 로그인 기능"
```

구현 작업을 단계별로 분해합니다.

**생성되는 내용:**
- Phase별 작업 구조
- 작업 간 의존성
- 우선순위
- 예상 소요 시간
- 검증 기준

### 5. 작업 실행

```bash
vibe run "Task 1-1"
```

특정 작업을 실행합니다.

### 6. 검증

```bash
vibe verify "사용자 로그인 기능"
```

구현 결과를 검증합니다.

## 실제 사용 예시

### 시나리오: TODO 앱 개발

**1단계: 명세 작성**

```bash
vibe spec "할 일 목록 관리"
```

**생성된 명세 (일부):**

```
When 사용자가 할 일을 입력하면
The system shall 데이터베이스에 저장한다

When 사용자가 완료 버튼을 누르면
The system shall 해당 항목을 완료 상태로 변경한다

Where 인터넷 연결이 없을 때
The system shall 로컬에 저장 후 동기화한다
```

**2단계: 계획 수립**

```bash
vibe plan "할 일 목록 관리"
```

**생성된 계획 (일부):**

```
기술 스택:
- Frontend: React 19
- Backend: FastAPI
- Database: PostgreSQL
- State: Zustand

아키텍처:
- RESTful API
- 낙관적 업데이트
- 오프라인 지원

예상 비용: $0 (오픈소스 스택)
```

**3단계: 작업 분해**

```bash
vibe tasks "할 일 목록 관리"
```

**생성된 작업 (일부):**

```
Phase 1: 데이터베이스
  Task 1-1: 스키마 설계
  Task 1-2: 마이그레이션 작성

Phase 2: 백엔드 API
  Task 2-1: CRUD 엔드포인트
  Task 2-2: 인증 미들웨어

Phase 3: 프론트엔드
  Task 3-1: UI 컴포넌트
  Task 3-2: 상태 관리
```

**4단계: 실행**

```bash
vibe run "Task 1-1"
```

AI가 스키마를 설계하고 코드를 생성합니다.

## 기존 방식 vs Vibe

### 기존 방식

```
1. 기획서 작성 (수동)
   ↓
2. 엔티티 추출 (수동)
   ↓
3. ERD 설계 (수동)
   ↓
4. API 설계 (수동)
   ↓
5. 작업 분해 (수동)
   ↓
6. AI에게 설명 (반복)
   ↓
7. 코드 생성 (AI)
   ↓
8. 검증 (수동)
```

**문제점:**
- 매번 처음부터
- 일관성 유지 어려움
- 시간 소모

### Vibe 방식

```
1. vibe spec "기능명"
   ↓
2. vibe plan "기능명"
   ↓
3. vibe tasks "기능명"
   ↓
4. vibe run "Task X-Y"
   ↓
5. vibe verify "기능명"
```

**장점:**
- 자동화된 프로세스
- 구조적 일관성
- 빠른 속도
