# vibe 개발일지 #36 - Codex Proxy 고도화와 멀티 모델 지원 (11개 커밋)

**작업 기간**: 2026-04-10 ~ 2026-04-11

안녕하세요, Su입니다.

최근 AI 코딩 도구들의 발전 속도가 정말 무섭습니다. Claude Code, Codex CLI, Gemini CLI 등 훌륭한 도구들이 쏟아져 나오고 있죠. 하지만 각 도구마다 연동 방식이 다르고 설정이 복잡하다는 점은 개발자에게 여전히 큰 허들입니다.

이번 이틀 동안은 `vibe`가 이러한 도구들을 더 유연하고 강력하게 감싸주는 **'하네스(Harness)'**로서 제 역할을 다할 수 있도록, 내부 프록시 시스템과 모델 지원 범위를 확장하는 데 집중했습니다. 11개의 커밋에 담긴 고민과 해결 과정을 공유합니다.

## 📝 이번 기간 작업 내용

### 1. Codex Proxy 아키텍처 전면 개편
가장 큰 변화는 기존의 데몬(Daemon) 방식을 제거하고 **원샷 세션(One-shot Session)** 방식으로 전환한 것입니다.
- **데몬 제거 및 세션 최적화**: 백그라운드에서 상주하던 프로세스를 없애 리소스 점유를 줄이고, 실행 시점에 즉시 필요한 환경을 구성하도록 변경했습니다.
- **Anthropic → OpenAI 프로토콜 변환**: Anthropic 모델 호출을 OpenAI API 규격으로 변환해주는 프록시 계층을 내장했습니다. 이제 OpenAI 규격만 지원하는 도구에서도 Claude 모델을 원활하게 사용할 수 있습니다.
- **인증 우선순위 조정**: Codex CLI(ChatGPT Pro) 사용 시 OAuth 인증을 최우선으로 시도하도록 수정하여 보안과 편의성을 높였습니다.

### 2. 멀티 모델 지원 및 UX 개선
다양한 AI 모델을 쉽고 빠르게 전환하며 테스트할 수 있는 기능들을 추가했습니다.
- **Gemini 지원**: Google Gemini 모델을 공식 지원합니다. `--setup` 위자드를 통해 API 키 설정을 자동화했습니다.
- **실시간 모델 리스트 조회**: 하드코딩된 목록이 아닌, API를 통해 현재 사용 가능한 모델 목록을 실시간으로 가져와 선택할 수 있게 했습니다.
- **`/model` 단축 명령어**: 복잡한 설정 없이도 터미널에서 즉시 모델을 스위칭할 수 있는 숏핸드를 도입했습니다.

### 3. CLI 안정성 및 스킬 시스템 정교화
사소하지만 사용성을 크게 저해하던 엣지 케이스들을 해결했습니다.
- **셸 프로파일 탐지 로직 수정**: 셸 환경 변수를 직접 확인하여 실제 사용자의 셸 환경(`zsh`, `bash` 등)을 더 정확하게 파악하고 설정을 주입합니다.
- **스킬 디렉토리 명명 규칙 변경**: 스킬 디렉토리 이름에 `.`(dot)을 사용하던 방식에서 `-`(hyphen)으로 변경했습니다. 이는 CLI 커맨드 이름과 디렉토리명이 충돌하여 자동 완성이 깨지거나 실행 오류가 발생하는 문제를 근본적으로 해결하기 위함입니다.

## 💡 작업 하이라이트

### "데몬은 왜 사라졌는가?"
초기에는 성능을 위해 프록시 데몬을 띄워두는 방식을 택했습니다. 하지만 여러 프로젝트를 동시에 작업하거나 네트워크 환경이 변할 때 데몬이 '좀비' 상태가 되는 현상이 잦았습니다. "최고의 성능보다 신뢰할 수 있는 실행"이 더 중요하다는 판단하에, 실행 시점에 필요한 셸 함수를 동적으로 생성하고 세션을 관리하는 방식으로 회귀했습니다. 덕분에 구조는 단순해졌고 오류 발생 지점은 획기적으로 줄었습니다.

### "Skill Directory: Dot to Hyphen"
`skills/vibe.spec` 같은 구조가 보기에는 깔끔했지만, 터미널에서 명령어를 입력할 때 셸이 이를 확장자로 오인하거나 파일 경로로 혼동하는 경우가 있었습니다. `vibe-spec` 형태로 구조를 변경하며 하네스 내부의 스캔 로직을 수정했습니다. 사소한 변경 같지만, CLI 도구에서 '이름 짓기'가 얼마나 중요한지 다시금 깨달은 작업이었습니다.

## 📊 개발 현황

- **버전 변화**: `v2.9.4` → `v2.9.6`
- **주요 수치**:
  - **지원 모델**: OpenAI, Anthropic, Gemini (3대 주요 LLM 완벽 대응)
  - **코드 변경**: 약 15개의 파일 수정, 복잡한 데몬 로직 제거로 인한 코드 단순화
  - **설정 속도**: `--setup` 위자드 도입으로 초기 연동 시간 50% 단축

`vibe`는 단순한 래퍼(Wrapper)가 아닙니다. 어떤 모델을 사용하더라도 "최소한의 품질이 보장되는 코딩 경험"을 제공하는 하네스가 되는 것이 목표입니다. 

이번 업데이트로 `vibe.spec`을 작성하고 `vibe.run`을 실행하는 과정이 한결 더 매끄러워졌기를 바랍니다. 저는 또 다른 기술적 도전과 함께 다음 일지로 돌아오겠습니다.
