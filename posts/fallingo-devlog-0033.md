## 💡 작업 하이라이트

**블로그 자동화 시스템 완전 재구성**

11월 10일, **블로그 자동화 시스템을 주간 기반에서 커밋 기반으로 완전히 전환**했습니다. 이는 우리가 지금 사용하고 있는 시스템입니다.

**핵심 변경 사항**:
1. **커밋 기반 수집**: 마지막 블로그 이후 커밋만 수집 (중복 방지)
2. **제목 및 파일명**: 커밋 기반으로 변경
3. **커밋 해시 범위**: 블로그 제목에 추가
4. **존댓말 사용**: 블로그 작성 스타일 명시
5. **프롬프트 개선**: 더 현실적인 내용 생성
6. **Dev Log 형식**: 개발 일지 형식으로 표준화

**GitHub Actions 권한 및 에러 처리**

태그 push 권한을 추가하고, GitHub API 업로드 에러 처리를 개선했습니다. 이는 자동화 시스템의 안정성을 높입니다.

**현재 작업 중인 시스템의 기반**

이 9개의 커밋이 바로 **지금 우리가 사용하는 블로그 자동 생성 시스템의 기반**입니다. 20개 커밋마다 개발일지를 작성하고, 존댓말을 사용하며, 커밋 해시를 포함하는 시스템이 여기서 완성되었습니다.

- **블로그 자동화**: 주간 기반 → 커밋 기반 완전 전환
