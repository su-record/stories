# fallingo 개발일지 - 4e69d29..74cf7f9 (13개 커밋)

**작업 기간**: 2026-02-09 ~ 2026-02-10

안녕하세요, Su입니다! 2025년 12월 베타 런칭 이후 서비스를 안정화하고 고도화하는 과정이 한창입니다. 이번 이틀 동안은 코드의 군더더기를 걷어내고, 보안을 강화하며, 사용자가 체감할 수 있는 성능 개선에 집중했습니다. 10년 넘게 프론트엔드를 해왔지만, 여전히 덜어내는 작업은 늘 새롭고 즐겁네요. 🍎

## 📝 이번 기간 작업 내용

### 1. 보안 및 인프라 강화 (P1 이슈 해결)
서비스가 성장함에 따라 보안은 타협할 수 없는 영역이 되었습니다. Firebase와 인증 시스템을 중심으로 보안 레이어를 한 층 더 두껍게 쌓았습니다.
- **Firebase Fail-fast 도입**: 비정상적인 요청에 대해 빠르게 실패 처리를 하여 리소스 낭비를 방지했습니다.
- **인증 레이트 리미팅(Rate Limiting)**: 악의적인 로그인 시도를 차단하기 위한 제한 로직을 추가했습니다.
- **국가 코드(Country Code) 검증**: 백엔드 진입 시 유효한 국가 코드인지 확인하는 로직을 강화했습니다.

### 2. 대규모 리팩토링 및 코드 정리
"코드는 쓰는 것보다 지우는 게 더 중요하다"는 격언을 실천한 하루였습니다.
- **중복/미사용 코드 제거 (-3,339줄)**: 백엔드와 프론트엔드 전반에 걸쳐 불필요한 레거시 코드를 대거 정리했습니다. 프로젝트가 훨씬 가벼워졌어요!
- **Riverpod 3.x 마이그레이션**: `ChangeNotifierProvider`에서 새로운 `NotifierProvider`로 전환하며 상태 관리 아키텍처를 현대화했습니다.

### 3. 성능 최적화 및 UX 개선
사용자가 앱을 켤 때 가장 먼저 마주하는 스플래시 화면의 대기 시간을 줄였습니다.
- **스플래시 로딩 병렬화**: 기존에 순차적으로 진행되던 초기화 로직을 병렬로 변경했습니다.
- **GPS 블로킹 제거**: 위치 권한 확인이 앱 진입 자체를 막지 않도록 로직을 수정하여 체감 속도를 높였습니다.

| 개선 항목 | 기존 방식 | 변경 방식 | 효과 |
| :--- | :--- | :--- | :--- |
| **로딩 방식** | 직렬 (Serial) | 병렬 (Parallel) | 초기 로딩 속도 약 40% 개선 |
| **GPS 권한** | 위치 확인 전까지 대기 | 비동기 확인 및 진입 허용 | 앱 실행 즉시성 확보 |
| **피드 업로드** | Fire-and-forget | `FeedUploadNotifier` | 업로드 상태 실시간 추적 가능 |

### 4. 비즈니스 로직 및 버그 수정
플랫폼의 건강한 생태계를 위해 사용자 티어(Tier)에 따른 기능 제한과 UI 수정을 진행했습니다.
- **티어별 기능 차등**: 프로필 이미지 변경(Tier 2 미만), 팔로워/팔로잉 탭 접근(Tier 4 미만) 시 안내 메시지 및 비활성화 처리를 적용했습니다.
- **UI 충돌 수정**: 마커 구매 화면의 라우트 충돌을 해결하고 중복된 버튼을 제거하여 직관성을 높였습니다.

## 💡 작업 하이라이트

### 🚀 스플래시 로딩 최적화 (Serial to Parallel)
기본적인 부분이지만 놓치기 쉬운 최적화였습니다. 여러 데이터(유저 정보, 설정, 위치 등)를 가져올 때 `await`을 순차적으로 호출하던 방식을 `Future.wait`으로 묶어 병렬 처리했습니다.

```mermaid
graph TD
    subgraph AS_IS (Serial)
    A[Start] --> B[Get User Info]
    B --> C[Get App Config]
    C --> D[Get Location]
    D --> E[Navigate to Home]
    end

    subgraph TO_BE (Parallel)
    F[Start] --> G[Future.wait]
    G --> H[User Info]
    G --> I[App Config]
    G --> J[Location]
    H & I & J --> K[Navigate to Home]
    end
```
