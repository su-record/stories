# vibe 개발일지 #40 - AI의 "선 넘기"를 방지하는 Scope Guard와 신뢰의 CI (10개 커밋)

**작업 기간**: 2026-04-14 ~ 2026-04-15

안녕하세요, 프론트엔드 개발자이자 AI 코딩 하네스 **vibe**를 만들고 있는 Su입니다.

최근 Claude Code나 Gemini CLI 같은 강력한 AI 에이전트들이 쏟아져 나오고 있죠. 하지만 에이전트가 강력해질수록 우리가 겪는 공포가 하나 있습니다. 바로 **"내 코드를 어디까지 건드리는 거지?"**라는 불안감입니다. 

vibe의 철학은 **"Agent = Model + Harness"**입니다. 모델이 똑똑해지는 건 우리가 어쩔 수 없지만, 그 모델이 안전하게 뛰어놀 수 있는 '하네스(Harness)'를 단단하게 만드는 건 개발자의 몫이죠. 이번 이틀간은 AI가 정해진 선을 넘지 못하게 만드는 **Scope Guard**와, 우리 도구 자체가 망가지지 않았음을 증명하는 **Self-test 시스템** 구축에 집중했습니다.

## 📝 이번 기간 작업 내용

### 🛡️ AI 가드레일: Scope Guard 시스템 강화
AI가 엉뚱한 파일을 수정해서 커밋을 엉망으로 만드는 것만큼 짜증 나는 일도 없죠. 이를 원천 차단하기 위한 장치들을 마련했습니다.

*   **PreToolUse 훅을 통한 수정 권한 강제**: AI가 `Edit`이나 `Write` 도구를 사용하기 직전에, 현재 정의된 `scope.json` 영역 밖의 파일을 건드리려 하면 실행을 즉시 차단합니다.
*   **Active SPEC 기반의 Scope 자동 도출**: 일일이 수정 범위를 명시하는 건 귀찮은 일입니다. 현재 활성화된 `/vibe.spec` 파일들을 분석해서 AI가 작업해야 할 범위를 자동으로 추출해 `scope.json`을 생성해 주는 기능을 추가했습니다.
*   **오탐(False Positive) 수정**: 파일 내용 중에 특정 패턴이 들어있다고 해서 수정 권한이 없다고 판단하던 버그를 수정했습니다. 이제는 정확히 '파일 경로'를 기준으로만 판단합니다.

### 🧪 신뢰 구축: Self-test 및 CI 파이프라인
vibe가 남의 코드를 고쳐주기 전에, vibe 자신부터 멀쩡해야 합니다.

*   **`/vibe.test` 및 `report` 서브 커맨드**: 배포된 모든 기능이 런타임에서 정상 작동하는지 전수 조사하는 테스트 사양을 추가했습니다.
*   **pnpm 10 대응 및 네이티브 빌드 최적화**: 최신 pnpm 환경에서 `better-sqlite3`의 네이티브 빌드가 막히던 문제를 해결했습니다. 로컬 DB 성능을 포기할 순 없으니까요.
*   **CI workflow 도입**: 이제 모든 PR과 푸시에서 빌드와 테스트가 필수(Required)로 통과되어야 합니다. "내 컴에선 되는데?"라는 변명은 이제 끝입니다.

### 🌐 편의성 및 세션 관리
*   **글로벌 협업 준비**: 이전 세션의 기록(.md)을 영어로 번역하는 기능을 실험적으로 추가했습니다.
*   **버전 릴리스**: 2.9.21부터 2.9.23까지 총 3번의 마이너 패치를 통해 안정성을 높였습니다.

## 💡 작업 하이라이트

### 1. "선 넘지 마세요" — Scope Guard의 진화
가장 의미 있었던 작업은 **PreToolUse 훅**의 도입입니다. 기존에는 AI에게 "이 파일들만 고쳐"라고 프롬프트로 부탁했다면, 이제는 **하네스 단에서 물리적으로 차단**합니다. 

```typescript
// 내부 로직의 핵심 개념
if (tool === 'write_file' && !isInScope(targetPath)) {
  throw new Error(`Access Denied: ${targetPath} is out of your current vibe scope.`);
}
```

이렇게 하면 AI가 아무리 똑똑하게 굴어도, 혹은 실수하더라도 프로젝트 전체가 오염되는 것을 막을 수 있습니다. "Minimum quality guaranteed"라는 vibe의 약속에 한 걸음 더 다가간 셈이죠.

### 2. CI/CD의 기본으로 돌아가기
혼자 개발하다 보면 테스트를 소홀히 하기 쉽습니다. 하지만 vibe는 다른 개발자들의 워크플로우에 깊숙이 관여하는 도구입니다. 그래서 `pnpm 10` 환경에서 발생한 빌드 병목을 해결하고, GitHub Actions를 통해 빌드와 테스트를 강제하도록 설정했습니다. 이제 `2.9.23` 버전은 그 어느 때보다 단단합니다.
