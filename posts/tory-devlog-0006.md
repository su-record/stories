# tory 개발일지 #6 - 코드 품질 + 보안 강화 + LLM 에이전트 9대 기능 (10개 커밋)

**작업 기간**: 2026-03-03

## 📝 이번 기간 작업 내용

### 버그 수정 + 분석 리포트 (2개 커밋)

codex/gemini CLI 래퍼의 버그와 TypeScript 컴파일 에러를 수정했다. 이후 전체 코드베이스에 대한 분석 리포트를 작성했다. 이 리포트가 이날 대부분 작업의 기준이 됐다.

| 커밋 | 내용 |
|------|------|
| `fix: codex/gemini CLI 래퍼 버그 수정 + TS 에러 해결` | CLI 래퍼 버그 및 타입 에러 수정 |
| `분석 리포트트` | 전체 코드베이스 분석 리포트 작성 |

### 코드 품질 개선 P1~P3 (3개 커밋)

분석 리포트를 기반으로 P1(우선순위 1)~P3까지 코드 품질 개선을 진행했다. 리팩터링으로 구조를 개선하고, 이후 보안과 안정성 이슈 6건을 수정했다. pnpm 버전 충돌도 이 시점에 해소됐다. DEL 커밋은 리포트 작성 중 생성된 임시 파일을 정리한 것이다.

| 커밋 | 내용 |
|------|------|
| `refactor: 분석 리포트 P1~P3 코드 품질 개선` | 코드 품질 리팩터링 P1~P3 |
| `DEL` | 임시 파일 삭제 |
| `fix: P1 보안·안정성 이슈 6건 수정 + pnpm 버전 충돌 해소` | 보안·안정성 버그 수정 및 의존성 충돌 해결 |

### 스킬 생태계 연동 (1개 커밋)

회의 합의 결과를 스킬로 자동 매핑하고 실행하는 파이프라인을 구현했다. `skills.sh` 생태계와의 연동으로, LLM 회의에서 "이 작업은 X 방식으로 처리하자"는 합의가 나오면 해당 스킬이 자동으로 실행된다. 회의가 결론에서 멈추지 않고 실행까지 이어지는 구조다.

| 커밋 | 내용 |
|------|------|
| `feat: skills.sh 생태계 연동 — 회의 합의 → 스킬 자동 매핑 → 실행` | 회의 합의 기반 스킬 자동 실행 파이프라인 |

### 보안 취약점 + 안정성 (2개 커밋)

`cargo audit`으로 발견된 취약점 19건에 대응했다. 즉시 패치가 가능한 항목은 업데이트하고, 현재 영향이 없는 항목은 `.cargo/audit.toml`에 ignore 설정을 추가해 의도적 수용을 명시했다. JWT_PRIVATE_KEY_PEM 파싱 실패 시 패닉 대신 HS256 폴백으로 전환해 서버가 키 문제로 죽지 않도록 했다.

| 커밋 | 내용 |
|------|------|
| `fix: cargo audit 취약점 19건 대응 — .cargo/audit.toml ignore 설정` | 의존성 취약점 대응 |
| `fix: JWT_PRIVATE_KEY_PEM 파싱 실패 시 패닉 대신 HS256 폴백` | JWT 키 파싱 오류 안전 처리 |

### API 엔드포인트 정리 + LLM 에이전트 9대 기능 (2개 커밋)

API_BASE 중복 선언을 제거하고 엔드포인트 URL을 통일했다. 마지막으로 LLM 에이전트 9대 기능을 한 번에 구현했다. Tool Calling, 웹 검색, 브라우저 자동화가 핵심이다.

| 커밋 | 내용 |
|------|------|
| `fix: API_BASE 중복 선언 제거 및 엔드포인트 URL 통일` | API 엔드포인트 URL 정리 |
| `feat: LLM 에이전트 9대 기능 구현 — Tool Calling, 웹 검색, 브라우저 자동화 외` | 에이전트 도구 실행 기능 9종 구현 |

## 💡 작업 하이라이트

**회의에서 실행까지, 자율성의 완성**

이날 작업의 핵심은 두 가지가 합쳐진 것이다. 스킬 연동과 LLM 에이전트 9대 기능.

스킬 연동은 회의의 결론을 실행으로 연결한다. LLM들이 브레인스토밍으로 "이 문제는 A 방식으로 해결하자"고 합의하면, Secretary가 합의 내용을 스킬 레지스트리에서 매핑하고 실행한다. 회의는 더 이상 조언으로 끝나지 않는다.

LLM 에이전트 9대 기능은 실행 범위를 확장한다. Tool Calling으로 외부 API를 호출하고, 웹 검색으로 최신 정보를 가져오고, 브라우저 자동화로 GUI 작업을 처리한다. 회의에서 합의된 작업이 실제로 무엇을 할 수 있는지의 범위가 이 기능들로 정해진다.

cargo audit 대응에서 `.cargo/audit.toml` ignore를 활용한 것은 보안 작업에서 중요한 실천이다. 모든 취약점을 패치할 수는 없다. 현재 코드 경로에서 실제로 도달 불가능한 취약점을 억지로 패치하려다 다른 문제가 생기는 것보다, 의도적으로 수용하고 문서화하는 것이 더 정직한 방법이다.
