# fallingo 개발일지 - e2ec42d..ea02088 (20개 커밋)

안녕하세요, Fallingo 개발자 Su입니다! 🖐️ 베타 런칭 목표일인 2025년 12월을 향해 정신없이 달려가고 있는 요즘입니다. 비전공자에 늦깎이 개발자라 항상 배움의 연속이지만, 이렇게 치열하게 무언가를 만들어가는 과정이 참 뿌듯하고 재미있어요. 이번 주도 지난 며칠 동안 어떤 삽질과 성장을 겪었는지 개발 일지로 공유해볼게요! 🚀

**작업 기간**: 2025-11-16 ~ 2025-11-18

## 📝 이번 기간 작업 내용

이번 기간에는 총 20개의 커밋을 통해 Fallingo의 핵심 기능들을 더욱 탄탄하게 다지는 데 집중했습니다. 특히 사용자 경험에 직결되는 소셜 기능과 AI 기반 검색 기능, 그리고 커뮤니티 활성화를 위한 랭킹 시스템 정비가 주된 작업이었어요.

### 1. 사용자 인증 및 소셜 기능 확장 (3개 커밋)
Fallingo는 결국 '소셜 플랫폼'인 만큼, 사용자들이 쉽고 편리하게 시작하고 소통할 수 있도록 만드는 게 정말 중요하다고 생각해요. 이번에 가장 큰 작업 중 하나는 바로 소셜 로그인 연동이었죠!

*   `feat: 소셜 로그인 및 회원가입 API 연동 완료`
*   `feat: 번역 API 및 회원가입 약관 확장`
*   `fix: 팔로우 시스템 정확하게 수정 - 일반 팔로우 버튼 존재`

드디어! 소셜 로그인 및 회원가입 API 연동을 완료했습니다. 🎉 역시나 예상했던 것보다 연동 과정이 복잡하고, 각 플랫폼별 정책 차이 때문에 애를 좀 먹었어요. 특히 보안과 개인 정보 처리에 신경 쓰느라 여러 번의 테스트를 거쳤습니다. 이제는 글로벌 유저들을 위해 다국어 번역 API를 연동하고 회원가입 약관도 확장해서, 어떤 사용자든 편하게 가입할 수 있도록 기반을 마련했어요. 기존 팔로우 시스템에 일반 팔로우 버튼이 정확히 작동하지 않던 버그도 수정해서 사용자들이 좀 더 직관적으로 관계를 맺을 수 있게 만들었습니다. 이번 작업으로 사용자 온보딩 경험이 훨씬 좋아질 것 같아요!

### 2. AI 기반 검색 및 위치 기능 고도화 (7개 커밋)
Fallingo의 핵심은 AI 기반의 '음식 탐색'이죠. 이 부분의 정확성과 효율성을 높이는 데 많은 시간을 투자했습니다.

*   `feat: 지오펜싱 알림 및 권한 관리 시스템 구현`
*   `fix: AI API 스팩 문서 완전 재작성 - 기획문서 기준`
*   `fix: 검색 API 통합 검색으로 단순화 (19개→12개)`
*   `fix: Gemini API 엔드포인트 정확하게 수정 (7개→2개)`
*   `fix: Vision API 통계 오버레이에 음식 메뉴 이름 추가`
*   `fix: 검색/탐색 API 엔드포인트 수정`

사용자 위치를 기반으로 한 지오펜싱(Geofencing) 알림 시스템과 권한 관리 시스템을 구현했습니다. 📍 이걸로 특정 지역에 진입하면 관련된 음식점 정보나 이벤트 알림을 줄 수 있게 되었죠. 개인적으로 FastAPI와 Flutter를 익히면서 가장 즐거웠던 부분 중 하나였습니다.

그리고 가장 반복적으로 수정했던 부분이 바로 AI API 스펙과 검색 API였습니다. 처음 기획 단계에서 생각했던 복잡한 검색 로직을 실제 개발 과정에서 통합하고 단순화하는 작업이 필요했어요. 🤯 19개나 되던 검색 API 엔드포인트를 12개로 줄였고, Gemini API 엔드포인트도 7개에서 2개로 대폭 정리했습니다. 또한 Vision API를 통해 음식 이미지를 분석하고 통계 오버레이에 정확한 메뉴 이름을 표시하도록 수정했어요. 기획 문서와 실제 구현 간의 간극을 줄이고, 더 효율적인 API 구조를 만드는 데 집중했습니다.

### 3. 커뮤니티 및 랭킹 시스템 정비 (6개 커밋)
Fallingo는 단순히 음식점을 찾는 것을 넘어, 미식가들이 서로 교류하고 성장하는 커뮤니티를 지향합니다. '대장금' 시스템과 티어 승급 로직을 대대적으로 손봤어요.

*   `fix: API 스팩 문서 대장금 선발 시스템 정확하게 수정`
*   `fix: 대장금 선발 시스템 완전 정리`
*   `fix: 대장금 선발 권역 구분 정확하게 수정`
*   `fix: 국가 승격 조건 완화 (MAU 10만→5만, 피드 50만→30만)`
*   `fix: 티어 승급 포인트 정확한 값으로 수정 (3.1배수 패턴)`
*   `refactor: '인증맛집' → '추천맛집' 용어 통일`

대장금 선발 시스템과 티어 승급 로직이 너무 복잡해서 개발하면서도 계속 헷갈렸어요. 😅 그래서 기획자와 여러 번 논의하며 API 스펙 문서를 재작성하고, 실제 로직도 완전히 정리했습니다. 특히 권역별 대장금 선발과 티어 승급 포인트(3.1배수 패턴)를 정확한 값으로 수정해서 시스템의 공정성을 높였어요. 또한, 서비스 초기에 유저들이 더 쉽게 국가를 승격시킬 수 있도록 조건을 완화(MAU 10만→5만, 피드 50만→30만)했습니다. 그리고 내부적으로 사용하던 '인증맛집'이라는 용어를 '추천맛집'으로 통일해서, 서비스의 지향점을 더욱 명확하게 보여주려고 노력했습니다.

### 4. 서비스 운영 및 프로덕션 준비 (2개 커밋)
베타 런칭이 다가오면서 안정적인 서비스 운영을 위한 준비도 빼놓을 수 없죠!

*   `feat: 프로덕션 준비 체크리스트 추가 + 인증 시스템 정리 완료`
*   `fix: Fallingo Official 자동 마케팅 시스템 추가`

프로덕션 배포 전에 빠짐없이 점검해야 할 항목들을 체크리스트로 정리하고, 그동안 쌓였던 인증 시스템 관련 코드들도 깔끔하게 정리했습니다. 그리고 Fallingo Official 계정을 통한 자동 마케팅 시스템을 추가해서, 런칭 후에도 활발한 정보 제공과 이벤트가 이루어질 수 있도록 기반을 마련했습니다.

### 5. 문서화 작업 (3개 커밋)
개발은 문서화의 연속이죠. 특히 API 스펙 변경이 많았던 만큼, 관련 문서 업데이트에 신경 썼습니다.

*   `docs: 통합 검색 type 파라미터 상세 설명 추가`
*   `docs: 통합 검색 우선순위 명확화`
*   `docs: 통합 검색 API 동작 방식 명확화`

통합 검색 API가 대폭 단순화되면서, 이에 대한 문서도 함께 업데이트했어요. `type` 파라미터의 상세 설명부터 검색 우선순위, 그리고 API의 전반적인 동작 방식을 명확하게 설명하는 문서를 추가했습니다. 저 혼자 하는 개발이지만, 나중에 다른 개발자가 합류하거나 제가 혹시 잊어버리더라도 헤매지 않도록 꼼꼼하게 정리하고 있어요. 📑

## 💡 작업 하이라이트

이번 기간의 가장 큰 하이라이트는 역시 **소셜 로그인 및 회원가입 API 연동 완료**와 **AI 기반 통합 검색 시스템의 대대적인 정비**였습니다. 사용자 온보딩의 핵심인 소셜 로그인을 통해 유입 장벽을 낮추고, Fallingo의 핵심 가치인 '음식 탐색'을 AI 기반으로 더욱 정확하고 효율적으로 만드는 데 큰 진전을 보였습니다.

반복된 `fix`와 `refactor` 커밋들이 많았던 이유는, 초기에 기획했던 이상적인 기능들이 실제 개발 과정에서 현실적인 제약이나 더 나은 사용자 경험을 위해 재고되고 있기 때문입니다. 특히 AI API 스펙이나 검색 로직은 기획 단계에서는 추상적으로 느껴지다가, 실제로 데이터를 연동하고 기능을 구현하면서 비효율적인 부분이 드러나는 경우가 많았어요. 😅 저처럼 새로운 기술 스택(FastAPI, Flutter)을 익히면서 진행하는 프로젝트다 보니, 시행착오가 많을 수밖에 없지만, 하나하나 해결해나가면서 배우는 게 정말 많습니다. 덕분에 훨씬 더 견고하고 효율적인 시스템을 만들고 있다고 믿어요! 💪
