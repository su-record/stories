# tory 개발일지 #26 - AI Team UI + Tory CLI (10개 커밋)

**작업 기간**: 2026-03-17 ~ 2026-03-18

## 📝 이번 기간 작업 내용

### AI Team 로스터 재구성 + 18개 프로필 (3개 커밋)

설정 탭의 AI Models 탭을 AI Team 로스터 기반으로 전면 재구성했다. 기존에는 모델 목록을 단순 나열하는 방식이었다면, 이번 개편으로 각 AI가 하나의 팀 멤버로 표현된다. 프로필을 기존 8개에서 18개로 확장하고, 특수 목적 모델 섹션도 별도로 마련했다. 18개 프로필 각각에 고유 SVG 아바타를 제작해 적용했다.

| 커밋 | 내용 |
|------|------|
| `feat(desktop): AI Models 탭 → AI Team 로스터 기반으로 재구성` | AI Team 탭 재구성 |
| `feat: 모델별 AI 프로필 확장 (8→18개) + 특수 목적 모델 섹션` | 프로필 18개 확장 |
| `feat(desktop): 18개 AI 프로필 고유 SVG 아바타 추가` | SVG 아바타 제작 |

### 프로바이더 자동 등록 + 프로필 카드 활성화 판단 (3개 커밋)

AppCore 시작 시 환경변수에서 API 키를 읽어 LLM 프로바이더를 자동으로 등록하도록 변경했다. CLI 키와 Keyring API 키도 함께 지원하며, CLI를 우선으로 처리한다. 프로필 카드의 활성화 여부를 `via` 필드 기반으로 판단하도록 수정했다. 특수 모델 섹션에서 Embedding 모델은 제거했다.

| 커밋 | 내용 |
|------|------|
| `fix(desktop): AppCore 시작 시 환경변수 API 키로 LLM 프로바이더 자동 등록` | 환경변수 자동 등록 |
| `fix(desktop): CLI 우선 + Keyring API 키로 프로바이더 자동 등록` | CLI/Keyring 우선순위 |
| `fix(desktop): 특수 모델 + 프로필 카드 활성화 판단을 via 필드 기반으로 수정` | 활성화 판단 수정 |
| `fix: Embedding을 Special Purpose Models에서 제거` | Embedding 제거 |

### 인증 영속화 + DB 정리 + Tory CLI (3개 커밋)

인증 세션을 로컬 SQLite에 저장해 재시작 후에도 재로그인이 필요 없도록 개선했다. Store v14 마이그레이션에서 서버 전용 테이블을 로컬 DB에서 제거하여 DB 스키마를 정리했다. 그리고 터미널에서 회의실을 직접 테스트할 수 있는 Tory CLI를 추가했다.

| 커밋 | 내용 |
|------|------|
| `fix(desktop): 인증 세션을 로컬 SQLite에 영속화 — 재로그인 불필요` | 인증 영속화 |
| `refactor(store): v14 마이그레이션 — 로컬 DB 정리 (서버 전용 테이블 제거)` | DB 스키마 정리 |
| `feat: Tory CLI — 터미널에서 회의실 테스트` | Tory CLI 추가 |

## 💡 작업 하이라이트

**18개 아바타, 18개 페르소나**

AI Team은 추상적인 모델 목록이 아니라 각자의 전문성과 성격을 가진 팀원 구성이다. 18개 프로필 각각에 SVG 아바타를 제작한 것은 이 방향성을 시각적으로 구체화하는 작업이었다. 아바타는 단순 장식이 아니라 사용자가 어떤 AI에게 무엇을 물어볼지 직관적으로 인식하는 데 도움을 준다.

**CLI로 회의실을 터미널에서**

Tory CLI는 GUI 없이 터미널에서 회의를 실행할 수 있는 도구다. 개발 중 엔진 동작을 빠르게 검증하거나 회의 결과를 스크립트로 자동화할 때 유용하다. 인증 세션 영속화와 맞물려, 한 번 로그인하면 이후 CLI 호출에서도 별도 인증 없이 바로 사용할 수 있다.
