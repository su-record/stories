# 왜 Fallingo를 만들게 됐나

2016년, 나는 구글에서 일하는 꿈을 꿨다.

갈 수 없는 꿈이었지만, 구글의 기술로 내 앱을 만들기로 했다.

그리고 2025년 5월, Fallingo를 만들기 시작했다.

## 가짜 리뷰를 없애고 싶었다

음식점 리뷰를 볼 때마다 의심했다.

**"이 사람 진짜 여기 갔을까?"**

- 별점 5개, 칭찬만 가득 → 사장님이 쓴 건 아닐까?
- 별점 1개, 악의적 비난 → 경쟁 업체가 쓴 건 아닐까?
- 사진은 인터넷에서 퍼온 것 같고
- 리뷰 내용은 복붙한 것 같고

**진짜 경험인지 알 수 없었다.**

그래서 생각했다.

> "실제로 그 장소에 가서, 음식을 먹은 사람만 리뷰를 쓸 수 있다면?"

## 3단계 검증 시스템

Fallingo는 세 가지로 검증한다.

**1. GPS 검증 (50m 이내)**
- PostGIS로 정확한 위치 확인
- 음식점 근처에 있어야만 피드 작성 가능

**2. Vision API 음식 감지**
- 카메라로 찍은 사진이 진짜 음식인지 AI가 확인
- 인터넷에서 퍼온 사진은 걸러냄

**3. OCR 영수증 검증**
- Document AI로 영수증 인식
- 24시간 이내 영수증만 인정
- 메뉴 아이템 자동 매칭

**모든 피드가 검증된 진짜 경험이다.**

## 팔로워가 아닌 노력으로

인스타그램이나 유튜브는 팔로워 수가 곧 영향력이다.

10만 팔로워가 추천하면 믿고, 100명 팔로워가 추천하면 무시한다.

**하지만 팔로워 수가 맛집을 판단하는 기준이 될 수 있을까?**

Fallingo는 다르다.

**11단계 티어 시스템:**
- Tier 1 "누룽지" → Tier 11 "전설의 대장금"
- 팔로워 수가 아닌 **검증된 포스팅 횟수**로 티어 상승
- 높은 티어일수록 영향력 범위 확대

노력하면 누구나 영향력 있는 미식가가 될 수 있다.

팔로워 수를 사거나, 바이럴 마케팅을 할 필요 없다.

**진짜 음식을 먹고, 진짜 리뷰를 쓰면 된다.**

## 왜 처음부터 글로벌인가?

Fallingo는 위치 기반 서비스다.

그리고 피드는 **실제로 음식을 먹을 때만** 생성된다.

- GPS 검증 (50m 이내)
- Vision API 음식 감지
- 영수증 OCR 검증

이 물리적 제약 때문에, 한 지역에서는 콘텐츠 생산이 제한적이다.

**로컬만으로는 부족하다:**
- 서울 100명 → 하루 3-5개 피드
- 사람들은 하루 1-2끼 외식
- 매번 포스팅하는 사람은 극소수
- 콘텐츠가 쌓이는 속도가 너무 느림

**글로벌로 가면 해결된다:**
- 서울 100명 + 도쿄 100명 + 뉴욕 100명 + ...
- 각 지역 하루 3-5개씩 → 전체로는 충분한 콘텐츠
- 14개 언어 지원으로 언어 장벽 제거
- 여행 가면 다른 나라 맛집도 바로 확인

**그래서 처음부터:**
- Google Translate API 14개 언어 자동 번역
- Google Cloud 글로벌 인프라
- 다국어 지원 UI/UX

위치 기반이지만, **글로벌 스케일로만 성공할 수 있는 서비스다.**

## 나의 꿈은 구글이니까

구글에 들어갈 순 없지만, 구글의 기술로 내 앱을 만들 수 있다.

**100% Google Cloud Platform:**
- Cloud Run (FastAPI 백엔드)
- Cloud SQL (PostgreSQL + PostGIS)
- Cloud Memorystore (Redis)
- Vision API (음식 감지)
- Gemini AI (리뷰 생성)
- Document AI (영수증 OCR)
- Translate API (14개 언어)
- Maps Platform (지도)
- Firebase (인증)

모든 것이 구글 기술이다.

백엔드도 모르고, Flutter도 처음이고, GCP도 처음이었다.

하지만 **AI와 함께라면 된다고 믿었다.**

그리고 실제로 그렇게 됐다.

- 글로벌로 맛집을 공유하는

진짜 음식 경험 플랫폼이다.

**나의 꿈은 구글이니까.**
