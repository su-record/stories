# 왜 fallingo를 만들게 됐나

2016년, 나는 구글에서 일하는 꿈을 꿨다.

갈 수 없는 꿈이었지만, 구글의 기술로 내 앱을 만들기로 했다.

그리고 2025년 5월, fallingo를 만들기 시작했다.

## 우리가 원하는 건 '맛집'이 아니다

"맛집을 찾았는데 정작 뭐를 먹어야 할지 모르겠어..." 이게 진짜 문제였습니다. 우리가 원하는 건 "강남 맛집 100선"이 아닙니다. 너무 멀어요. "이 식당이 유명해요"도 아닙니다. 뭘 먹어야 하는데요? 우리가 정말 원하는 것은 "지금 갈 수 있는 거리의 맛있는 메뉴"입니다.

기존 앱들은 점포 중심이었습니다. 식당이 메인이고, 메뉴는 부가 정보이며, 리뷰는 식당 전체에 대해 작성됩니다. 하지만 fallingo는 메뉴 중심입니다. 메뉴가 메인이고, 식당은 메뉴가 있는 곳이며, 리뷰는 특정 메뉴에 대해 작성됩니다. 이 작은 차이가 사용자 경험을 완전히 바꿉니다.

## 가짜 리뷰를 없애고 싶었다

그리고 또 하나의 문제가 있었습니다. "이 사람 진짜 여기 갔을까?" 별점 5개에 칭찬만 가득한 리뷰를 보면 사장님이 쓴 건 아닐까 의심스럽습니다. 별점 1개에 악의적 비난이 담긴 리뷰는 경쟁 업체가 쓴 건 아닐까 싶고요. 사진은 인터넷에서 퍼온 것 같고, 리뷰 내용은 복붙한 것 같습니다. 진짜 경험인지 알 수 없었습니다.

그래서 생각했습니다. "실제로 그 장소에 가서, 음식을 먹은 사람만 리뷰를 쓸 수 있다면?"

## 3단계 검증 시스템

fallingo는 세 가지 방식으로 검증합니다. 첫째, GPS 검증입니다. PostGIS로 정확한 위치를 확인하고, 음식점 50m 이내에 있어야만 피드를 작성할 수 있습니다. 둘째, Vision API 음식 감지입니다. 카메라로 찍은 사진이 진짜 음식인지 AI가 확인하고, 인터넷에서 퍼온 사진은 걸러냅니다. 셋째, OCR 영수증 검증입니다. Document AI로 영수증을 인식하고, 24시간 이내 영수증만 인정하며, 메뉴 아이템을 자동으로 매칭합니다.

모든 피드가 검증된 진짜 경험입니다.

## 팔로워가 아닌 노력으로

인스타그램이나 유튜브는 팔로워 수가 곧 영향력입니다. 10만 팔로워가 추천하면 믿고, 100명 팔로워가 추천하면 무시합니다. 하지만 팔로워 수가 맛집을 판단하는 기준이 될 수 있을까요? fallingo는 다릅니다.

**11단계 티어 시스템:**

| 티어 | 이름 | 필요 포인트 | 영향력 범위 | 해금 기능 |
|------|------|------------|-----------|----------|
| 1 | 누룽지 | 0 | 5km | 기본 피드 작성 |
| 2 | 비빔밥 | 100 | 10km | 프로필 이미지 |
| 3 | 삼겹살 | 200 | 20km | 닉네임 설정 |
| 4 | 갈비탕 | 500 | 30km | 팔로우 기능 |
| 5 | 떡볶이 | 1,000 | 50km | 댓글 작성 |
| 6 | 짜장면 | 2,000 | 100km | 좋아요 기능 |
| 7 | 된장찌개 | 5,000 | 200km | 북마크 기능 |
| 8 | 불고기 | 10,000 | 500km | 메시지 기능 |
| 9 | 한정식 | 20,000 | 1,000km | 추천 알고리즘 우선순위 |
| 10 | 대장금 | 50,000 | 무제한 | 전체 지역 영향력 |
| 11 | 대장금⭐⭐⭐ | 100,000 | 무제한 | 전설의 미식가 (전 세계 100-200명) |

포인트는 검증된 피드를 작성하면 10포인트를 받고, 인기 피드는 좋아요와 댓글이 많으면 추가 보너스를 받습니다. 다만 허위 피드로 신고를 당하면 50포인트가 감점됩니다.

누룽지에서 대장금⭐⭐⭐까지 가는 여정은 팔로워 수가 아닌 검증된 포스팅 횟수로 티어가 상승합니다. 100,000 포인트는 10,000개의 검증된 피드를 의미하는데, 하루 1개씩 올린다면 약 27년이 걸립니다. 전 세계에서 100-200명만 도달하는 전설의 티어입니다. 높은 티어일수록 영향력 범위가 5km에서 무제한으로 확대됩니다.

노력하면 누구나 영향력 있는 미식가가 될 수 있습니다. 팔로워 수를 사거나, 바이럴 마케팅을 할 필요가 없습니다. 진짜 음식을 먹고, 진짜 리뷰를 쓰면 됩니다.

## 왜 처음부터 글로벌인가?

fallingo는 위치 기반 서비스입니다. 그리고 피드는 실제로 음식을 먹을 때만 생성됩니다. GPS 검증으로 50m 이내에 있어야 하고, Vision API로 음식을 감지하며, 영수증 OCR로 검증합니다. 이런 물리적 제약 때문에 한 지역에서는 콘텐츠 생산이 제한적입니다.

로컬만으로는 부족합니다. 서울에 100명이 있다면 하루 3-5개 피드가 생성됩니다. 사람들은 하루 1-2끼 외식을 하고, 매번 포스팅하는 사람은 극소수이기 때문에 콘텐츠가 쌓이는 속도가 너무 느립니다.

글로벌로 가면 해결됩니다. 서울 100명, 도쿄 100명, 뉴욕 100명이 각각 하루 3-5개씩 피드를 올리면 전체로는 충분한 콘텐츠가 됩니다. 14개 언어 지원으로 언어 장벽을 제거하고, 여행을 가면 다른 나라 맛집도 바로 확인할 수 있습니다.

**9개 지역 네트워크 구조:**
1. 🇰🇷 Korea (개별 국가)
2. 🇯🇵 Japan & Southeast Asia (일본, 베트남, 태국, 필리핀, 인도네시아, 말레이시아, 싱가포르)
3. 🇨🇳 China & Taiwan (중국, 대만, 홍콩, 마카오)
4. 🇮🇳 India & South Asia (인도, 파키스탄, 방글라데시, 스리랑카)
5. 🇺🇸 USA & Canada (미국, 캐나다)
6. 🇬🇧 Europe (영국, 프랑스, 독일, 이탈리아, 스페인 등)
7. 🇦🇺 Oceania (호주, 뉴질랜드)
8. 🇧🇷 Latin America (브라질, 멕시코, 아르헨티나 등)
9. 🌍 Africa & Middle East (남아공, UAE, 사우디 등)

**14개 언어 지원:**
- 한국어, 영어, 일본어, 중국어(간체/번체)
- 베트남어, 태국어, 인도네시아어, 말레이어
- 힌디어, 스페인어, 프랑스어, 독일어, 아랍어

**그래서 처음부터:**
- Google Translate API 14개 언어 자동 번역
- Google Cloud 글로벌 인프라
- 다국어 지원 UI/UX
- 지역별 음식 문화 데이터셋

위치 기반이지만, **글로벌 스케일로만 성공할 수 있는 서비스다.**

## 나의 꿈은 구글이니까

구글에 들어갈 순 없지만, 구글의 기술로 내 앱을 만들 수 있다.

**100% Google Cloud Platform:**
- Cloud Run (FastAPI 백엔드)
- Cloud SQL (PostgreSQL + PostGIS)
- Cloud Memorystore (Redis)
- Vision API (음식 감지)
- Gemini AI (리뷰 생성)
- Document AI (영수증 OCR)
- Translate API (14개 언어)
- Maps Platform (지도)
- Firebase (인증)

모든 것이 구글 기술이다.

백엔드도 모르고, Flutter도 처음이고, GCP도 처음이었다.

하지만 **AI와 함께라면 된다고 믿었다.**

그리고 실제로 그렇게 됐다.

## 맛에 빠져들어, 함께 떠나는 여정

**Falling in, Go together.**

fallingo는 단순히 맛집을 찾는 앱이 아니다.

**진짜 음식 경험을 공유하는 플랫폼이다:**

| 기존 앱 | fallingo |
|---------|----------|
| 점포 중심 | 메뉴 중심 |
| 누구나 작성 | 실제 방문자만 |
| 팔로워 수 = 영향력 | 노력 = 영향력 |
| 허위 정보 | 3중 검증 |
| 로컬 한정 | 글로벌 9개 지역 |
| 번역 없음 | 14개 언어 자동 번역 |

**세 가지 사용자 가치:**

1. **일반 사용자**: 3단계 검증으로 믿을 수 있는 맛집 정보
2. **미식가**: 누룽지에서 대장금⭐⭐⭐까지 성장하는 RPG 같은 경험
3. **소상공인**: 비싼 광고비 없이도 공정하게 노출되는 기회

**나의 꿈은 구글이니까.**

구글에 들어갈 순 없지만, 구글의 기술로 내 꿈을 만들 수 있다.

fallingo는 그렇게 시작됐다.
