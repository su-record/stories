# vibe 개발일지 #3 - 하네스 시스템의 완성 그리고 Figma 파이프라인의 혁신 (276개 커밋)

**작업 기간**: 2026-02-19 ~ 2026-04-09

안녕하세요, Su입니다. 지난 2월 첫 커밋을 시작으로 정신없이 달려온 `vibe` 프로젝트가 어느덧 v2.8.52 버전에 도달했습니다. 270개가 넘는 커밋을 쌓아오며 제가 집중했던 것은 단 하나입니다. **"어떻게 하면 AI가 만드는 코드의 품질을 하네스(Harness) 시스템으로 보장할 것인가?"**

이번 일지에서는 그동안의 치열했던 고민과 기술적 도전들을 정리해 보았습니다.

## 📝 이번 기간 작업 내용

### 1. 하네스(Harness) 시스템 및 아키텍처 정립
"에이전트 = 모델 + 하네스"라는 저만의 철학을 코드베이스에 완전히 녹여냈습니다.
- **Harness-100 패턴 적용**: 에이전트 팀, 오케스트레이터, 도메인 프레임워크를 구조화하여 AI가 길을 잃지 않게 설계했습니다.
- **Self-Healing 시스템**: `code-check` 훅이 단순히 에러를 차단하는 것을 넘어, 자동으로 수정안을 제안하는 기능을 추가했습니다.
- **LLM 비용 및 품질 추적**: `/vibe stats` 명령을 통해 세션별 비용 추적과 품질 트렌드를 리포트화했습니다.

### 2. Figma-to-Code 파이프라인 혁신 (`vibe.figma`)
단순히 디자인을 보고 코드를 짜는 수준을 넘어, Kombai 수준의 정교한 변환을 목표로 했습니다.
- **MCP 의존성 제거**: 외부 MCP 플러그인 대신 Figma REST API를 직접 다루는 자체 도구(`src/infra/lib/figma`)로 전면 전환하여 안정성을 높였습니다.
- **트리 기반 구조적 매핑**: 스크린샷 기반의 추정 방식에서 탈피하여, Figma의 레이어 트리를 분석해 HTML 구조와 매핑하는 '퍼즐 조립' 방식을 도입했습니다.
- **E2E 브라우저 검증**: Puppeteer와 CDP를 연동해 AI가 만든 UI가 실제 브라우저에서 디자인과 일치하는지 시각적으로 검증하는 루프를 추가했습니다.

### 3. 모델 오케스트레이션 및 인프라 최적화
- **Dynamic Routing**: GPT-5.4(추론), GPT-5.3-Codex(코딩), Gemini 3.1 Flash(이미지/대량 컨텍스트) 등 각 작업에 최적화된 모델을 동적으로 할당합니다.
- **OAuth 제거 및 CLI 통합**: 복잡한 인증 과정을 빼고 Codex CLI, Gemini CLI와 직접 연동하여 사용성을 극대화했습니다.
- **스킬 시스템 슬림화**: Karpathy의 가이드라인을 참고하여 방대해진 17개 스킬 파일을 최적화해 약 3,000줄의 불필요한 코드를 덜어냈습니다.

## 💡 작업 하이라이트

### "CSS Art 금지" - 실무 지향적 제약 조건 (Hard Rule)
초기 버전의 `vibe.figma`는 AI가 복잡한 그래픽 요소까지 CSS로 그리려다 보니 코드가 지저분해지는 문제가 있었습니다. 이를 해결하기 위해 **"디자인 에셋은 무조건 추출하고, CSS로 재현하지 않는다"**는 강력한 룰을 스킬에 심었습니다. 덕분에 코드는 깔끔해졌고, 실제 서비스에 바로 배포 가능한 수준의 결과물이 나오기 시작했습니다.

### 브라우저 기반의 시각적 피드백 루프
AI가 코드를 짠 후 "다 됐습니다"라고 말하는 것만으로는 부족했습니다. v2.8에서는 AI가 스스로 Puppeteer를 띄워 화면을 렌더링하고, 디자인 스토리보드와 비교하여 틀린 부분을 스스로 수정하는 **Verification Loop**를 구현했습니다. 하네스가 AI의 '눈' 역할을 하게 된 셈이죠.

## 📊 개발 현황

- **최적화 지표**:
    *   불필요한 레거시 코드 **-4,500줄 이상** 삭제
    *   활성화된 자동화 스킬: **45개**
    *   지원 모델: GPT-5.4 series, Gemini 3.1 series
- **자동화의 결실**: 오늘 이 개발일지조차 제가 만든 `devlog` 스킬과 GitHub Actions, Gemini를 통해 초안이 자동 생성되었습니다. (물론 마지막 검수는 제가 직접 정성을 담아 했습니다!)
