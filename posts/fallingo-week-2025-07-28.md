## 이번 주 요약

Reset 이후 2주간 새로운 방법론을 정립하고 재시작을 준비했다. 기획 문서를 다시 검토하며 누락된 내용을 발견했고, SpecKit이라는 체계적인 프레임워크를 알게 되었다. 서두르지 않고 탄탄한 기반을 만드는 데 집중했다.

## 주요 작업

### 기획 문서 재검토

**누락된 내용 발견**
- 티어별 상호작용 반경의 구체적인 수치
- 포인트 획득/차감 규칙의 세부 조건
- 음식점 정보 수집 방식과 데이터 소스
- 알림 시스템의 상세 요구사항
- 관리자 기능 정의

**개선한 내용**
- 각 기능의 우선순위 명확화
- 사용자 시나리오를 더 구체적으로 작성
- 예외 상황과 엣지 케이스 추가
- 성능 요구사항 정의

### SpecKit 도입 검토

**SpecKit이란**
- AI-First 개발을 위한 체계적인 프레임워크
- 스펙 → 플랜 → 태스크 → 구현의 명확한 흐름
- 문서 간 일관성 유지를 위한 검증 도구
- Constitution을 통한 프로젝트 원칙 정의

**마이그레이션 계획**
- 기존 기획 문서를 spec.md로 변환
- 엔티티 추출을 plan.md로 체계화
- 구현 작업을 tasks.md로 관리
- Constitution에 개발 원칙과 규칙 정의

**기대 효과**
- 문서 간 의존성 관리 자동화
- 일관성 검증 프로세스 내장
- AI가 따라야 할 명확한 가이드라인
- 작업 진행 상황 추적 용이

### AI-First 개발 방법론 정리

지난 몇 달간의 경험을 바탕으로 내가 찾은 방법론을 정리했다:

**핵심 원칙**

1. **문서가 곧 코드다**
   - 문서의 품질이 코드의 품질을 결정
   - 일관성 있는 문서 = 일관성 있는 코드
   - AI는 문서를 읽고 코드를 작성

2. **순서를 지켜라**
   - 각 단계는 독립적이지 않고 연결됨
   - 이전 단계의 산출물이 다음 단계의 입력
   - 건너뛰거나 동시에 하면 불일치 발생

3. **검증, 검증, 검증**
   - 한 번의 작성으로 완벽할 수 없음
   - 교차 검증을 통한 품질 확보
   - 문제를 빨리 발견할수록 수정 비용 감소

4. **AI를 의심하라**
   - AI는 "그럴싸한" 것을 잘 만듦
   - 하지만 그럴싸함과 정확함은 다름
   - 요구하지 않은 기능 추가를 경계

5. **백엔드부터 탄탄하게**
   - 데이터 구조가 모든 것의 기반
   - API가 확정되어야 프론트엔드 개발 가능
   - 동시 개발은 불일치를 만듦

**실천 방법**

- 각 세션마다 Constitution과 이전 산출물 제공
- "기획서를 순서대로 읽을 것" 명시
- 새로운 기능 추가 금지, 문서에 있는 것만 구현
- 각 단계 완료 후 품질 검증 요청
- 차이점 발견 시 원인 분석 후 재작성

## 배운 점 / 고민

**서두르지 않는 용기**

이전에는 빨리 결과물을 보고 싶어서 서둘렀다. 문서 작성은 대충하고 개발부터 시작했다. 하지만 그 결과는 Reset이었다.

이번에는 다르다. 2주 동안 한 줄의 코드도 작성하지 않았다. 하지만 이 2주가 앞으로 몇 달을 결정할 것이다. 탄탄한 기반 위에서 빠르게 갈 수 있다.

**AI와 함께 성장하기**

AI를 도구로만 생각했다. "이걸 만들어줘"라고 요구하면 만들어주는 도구. 하지만 AI와의 협업은 그보다 복잡하다.

- 어떻게 물어봐야 원하는 답을 얻는지
- 어떤 정보를 제공해야 일관성이 유지되는지
- 어떤 지침을 주어야 실수를 방지할 수 있는지

이런 것들을 배우는 과정이었다. AI-First 개발은 새로운 스킬셋이 필요한 새로운 패러다임이다.

**실패는 학습이다**

Reset은 실패가 아니라 학습이었다. 지난 몇 달간:
- AI의 한계와 가능성을 이해했다
- 문서의 중요성을 체감했다
- 올바른 프로세스를 찾았다
- 버리는 용기를 배웠다

이제 더 나은 방법으로 다시 시작할 준비가 되었다.

**2주간 한 줄의 코드도 쓰지 않았다**

2주 동안 한 줄의 코드도 작성하지 않았다. 기획 문서를 다시 검토하고, SpecKit을 공부하고, 방법론을 정리했다. 서두르지 않는 것이 용기라는 것을 배웠다. 이 2주가 앞으로 몇 달을 결정할 것이다. 탄탄한 기반 위에서 빠르게 갈 수 있다. 이번에는 다르다. 천천히, 확실하게, 올바르게.
