# tory 개발일지 #3 - 자율 토론 엔진 재설계와 캐릭터 오버레이 시스템 구축 (10개 커밋)

## 💡 작업 하이라이트

**버스(Bus) 기반 자율 토론 엔진으로의 아키텍처 전환**

기존 엔진은 에이전트의 발언 순서와 데이터 흐름이 고정되어 있어 유연한 확장이 어려웠습니다. 이번 업데이트에서는 모든 인지 프로세스를 중앙 메시지 버스에 연결했습니다. 이제 각 LLM 에이전트는 버스에서 발생하는 이벤트를 구독하고, 자신의 역할에 맞는 시점에 자율적으로 개입할 수 있습니다. 이는 특히 Quick(병렬조사)이나 Freeform(자율토론) 프로토콜에서 에이전트들이 동시다발적으로 정보를 처리하고 상호 비판하는 과정을 더욱 자연스럽게 만듭니다. Rust의 강력한 타입 시스템과 결합하여 동시성 제어 문제를 해결하면서 성능 최적화를 달성했습니다.

**Tauri 2를 활용한 데스크톱 캐릭터 레이어 구현**

사용자와 항상 함께하는 비서실장의 느낌을 주기 위해 단순한 창 형태의 앱을 넘어선 '오버레이' 시스템을 구축했습니다. Tauri 2의 멀티 윈도우 기능을 활용하여 캐릭터가 데스크톱 화면의 최상단에 떠 있도록 설정했습니다. 기술적 핵심은 투명한 배경 처리와 상황에 따른 입력 제어입니다. 캐릭터가 반응할 필요가 없을 때는 마우스 클릭이 캐릭터를 관통하여 뒤쪽의 작업 앱에 전달되도록 처리했고, 사용자가 대화를 시도할 때만 입력을 활성화하는 동적 제어 로직을 적용했습니다. 이는 업무 몰입도를 저해하지 않으면서도 필요할 때 즉각 호출 가능한 AI 에이전트의 사용자 경험을 완성하는 핵심 요소입니다.

**시각적 정체성 및 UX 안정화**

다수의 AI가 참여하는 회의 특성상 발언 주체를 명확히 구분하는 것이 중요했습니다. 각 에이전트에게 고유한 동물 캐릭터 아바타를 부여하고, 이를 서버 API와 연동하여 동적으로 로드하게 했습니다. 또한 UI 리팩토링 과정에서 발생한 CSS 레이어 간섭 문제와 투명도 처리 오류를 수정했습니다. 특히 투명 배경 설정이 전체 UI의 렌더링에 영향을 주지 않도록 스타일 시트 구조를 재정비하여 시각적 완성도를 높였습니다.
