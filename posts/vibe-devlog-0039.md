# vibe 개발일지 #39 - 계약(Contract)으로 묶고 회귀(Regression)로 진화하기 (11개 커밋)

**작업 기간**: 2026-04-13 ~ 2026-04-14

안녕하세요, Su입니다. 

지난 이틀간은 **vibe** 프로젝트의 핵심 철학인 "최소한의 품질 보장(Minimum quality guaranteed)"을 시스템적으로 한 단계 더 끌어올리는 작업을 진행했습니다. 단순히 AI가 코드를 잘 짜길 기도하는 게 아니라, 시스템이 코드를 감시하고 스스로 진화할 수 있는 장치들을 마련했는데요. 이번 업데이트의 핵심은 **계약(Contract)**과 **회귀(Regression)**입니다.

## 📝 이번 기간 작업 내용

이번 11개의 커밋은 크게 네 가지 영역으로 나눌 수 있습니다.

### 1. 에이전트 신뢰성 강화: `/vibe.regress` & `/vibe.contract`
가장 큰 변화는 새로운 실행 스펙의 추가입니다. AI가 코드를 수정할 때 기존 기능이 망가지는 것을 방지하고, 설계 단계의 약속이 유지되는지 확인하는 기능을 넣었습니다.
* **회귀 자동 진화**: `/vibe.regress` 명령어를 통해 에이전트가 이전의 성공 케이스를 바탕으로 스스로 테스트를 강화합니다.
* **계약 드리프트 감지**: `/vibe.contract`를 추가해, 코드가 수정되면서 API 명세나 타입 정의에서 멀어지는 현상(Drift)을 자동으로 포착합니다.

### 2. 리서치 프로세스 최적화 및 영구화
AI가 코드를 짜기 전 수행하는 '탐색' 과정의 결과물을 버리지 않고 자산화하는 작업을 했습니다.
* **병렬 리서치 결과 영구화**: `parallel-research` 결과를 `.claude/vibe/research/` 디렉토리에 저장하여 나중에 다시 참고할 수 있게 했습니다.
* **캐시 재사용**: `/vibe.spec` 단계에서 생성된 리서치 캐시를 활용해 중복된 연산을 줄였습니다.

### 3. 인프라 안정화 (SQLite 이슈 해결)
개발 환경에서 겪었던 뼈아픈 버그 하나를 잡았습니다.
* **`better-sqlite3` 해석 오류 수정**: `~/.vibe` 폴더로 `dist-only` 복사본을 옮길 때, 네이티브 모듈인 SQLite 라이브러리를 제대로 찾지 못하던 문제를 해결했습니다. 런타임 환경에 따라 의존성을 올바르게 해석하도록 로직을 보완했습니다.

### 4. 문서화 및 개념 정립
vibe가 지향하는 방향성을 문서로 명확히 했습니다.
* **Context Portability의 진화**: 단순히 '맥락을 옮긴다'는 선언적 의미에서 벗어나, 하네스 중립적인(harness-neutral) **'계약 기반 명문화'**로 개념을 수정했습니다.
* **에이전트 대상 정리**: `/vibe.docs` 대상에서 불필요해진 `GEMINI.md`를 제거하여 유지보수 효율을 높였습니다.

## 💡 작업 하이라이트

### "Agent = Model + Harness"의 실체화
제가 늘 강조하는 철학이지만, 모델(Claude, Gemini 등)이 똑똑해지는 것만으로는 부족합니다. 이번에 추가한 `/vibe.contract`는 모델이 '멋대로' 인터페이스를 바꾸지 못하게 막는 강력한 허들입니다.

**Before:**
- 에이전트가 기능을 구현하면서 은근슬쩍 API 응답 필드 이름을 바꿈.
- 런타임 에러가 발생해야만 문제를 인지함.

**After:**
- `/vibe.contract` 실행 시, 최초 정의된 Spec과 현재 코드의 괴리를 체크.
- 계약 위반 시 즉시 에러를 발생시키고 수정을 요구함.

### 연구 결과의 자산화 (Research Persistence)
AI 코딩 도구들이 가장 아쉬운 점 중 하나가 "방금 조사한 걸 왜 또 물어보지?" 하는 점이었습니다. 이번 업데이트로 `.claude/vibe/research/` 아래에 리서치 결과가 쌓이게 됩니다. 이는 단순한 로그가 아니라, 다음 실행 시 `/vibe.spec`이 참고할 수 있는 지식 베이스가 됩니다.

## 📊 개발 현황

- **버전 변화**: `v2.9.17` → `v2.9.20`
- **주요 추가 명령어**:
  - `/vibe.regress`: 회귀 테스트 자동 생성 및 검증
  - `/vibe.contract`: API 및 타입 계약 준수 확인
- **최적화**:
  - 리서치 단계 캐시 도입으로 평균 실행 속도 약 15% 향상 기대
  - `better-sqlite3` 로딩 이슈 해결로 설치 안정성 확보

이번 작업은 단순히 기능을 추가하는 것을 넘어, **'AI가 믿고 맡길 수 있는 동료가 되려면 어떤 제약 조건이 필요한가'**에 대한 제 나름의 답을 코드로 풀어낸 과정이었습니다. 특히 SQLite 관련 버그를 잡으면서 "역시 네이티브 모듈은 만만치 않구나"라는 배움도 얻었네요. (웃음)
