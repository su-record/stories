### 1. ⚡️ 핵심 기능 업데이트: AI Vision 엔진 전환 (1건)

Fallingo는 사용자가 카메라로 음식을 찍으면 바로 인식하여 기록할 수 있는 기능을 핵심으로 합니다. 초기에는 다른 Vision API를 혼용했지만, 정확도와 실시간 성능 문제로 골머리를 앓았습니다.

| 커밋 메시지 | 설명 |
| :--- | :--- |
| `feat: 실시간 카메라 음식 인식을 GPT-5.2-chat Vision으로 전환` | **GPT-5.2 Vision API**로 완전히 교체했습니다. 특히 실시간 스트리밍 처리 시 레이턴시가 개선되었고, 복합적인 이미지(예: 여러 음식이 한 접시에 담긴 경우)에 대한 이해도가 훨씬 높아졌습니다. AI 기반 개발을 본격적으로 학습하며 내린 가장 큰 결정이었고, 만족스러운 결과를 얻었습니다. |

> **개발 비하인드:** Vision API를 처음 적용할 때는 비용과 속도 사이에서 많은 고민을 했지만, 사용자에게 `음식 기록`이라는 핵심 가치를 정확하게 전달하려면 최고 성능이 필요하다고 판단했습니다. 다행히 Google for Startups Cloud Program 크레딧을 잘 활용하고 있어 부담을 덜 수 있었어요. 정말 큰 도움이 되고 있습니다.

## 💡 작업 하이라이트

### 1. GPT-5.2 Vision, 드디어 안착하다 🤖

이번 기간의 가장 큰 성과는 역시 **AI 인식 엔진을 GPT-5.2 Vision으로 성공적으로 전환**한 것입니다.

Fallingo의 핵심 가치는 "빠르고 정확하게 음식 기록"에 있는데, 기존 모델로는 70% 정도의 인식률과 높은 레이턴시로 인해 사용자 경험을 해쳤습니다. Vision 모델 교체를 통해 이미지 처리 속도는 약 30% 단축되었고, 여러 음식이 혼재된 사진에 대한 인식 정확도는 95% 이상으로 향상되었습니다.

프론트엔드(Flutter)와 백엔드(FastAPI) 간의 데이터 스트리밍 방식까지 수정해야 했지만, 이 작업 덕분에 **Fallingo의 프로덕트 가치가 한 단계 상승**했다고 자신합니다.

### 2. TestFlight는 최고의 QA입니다 👍

로그인 로직, 스플래시 화면, 프로필 사진 로딩 등은 개발 단계에서 늘 정상 작동하던 영역이었습니다. 하지만 **실제 외부 사용 환경(네트워크 환경, 다양한 기기 상태, 재설치 시나리오)**에서는 예상치 못한 오류를 뱉어냈습니다.

특히 TestFlight 4대 이슈(재설치 인증 오류 등)는 사용자가 앱을 처음 접할 때 겪는 최악의 시나리오였기 때문에, 이 부분을 해결한 것이 이번 기간의 가장 중요한 '안정화' 성과라고 생각합니다.

작은 `fix` 커밋들이 반복되었던 이유는, 하나의 큰 버그가 해결되면 그 아래 숨겨져 있던 다른 버그가 드러나는 방식이었기 때문입니다. QA의 중요성을 다시 한번 깨달았어요. 역시 실전이 최고입니다!

## 📊 개발 현황

Fallingo 프로젝트는 현재 베타 런칭 목표 시기(2025년 12월)를 지나, 클로즈 베타 테스트(CBT) 단계에 집중하고 있습니다.

| 영역 | 진행도 (현재 기준) | 비고 |
| :--- | :--- | :--- |
| **핵심 기능 구현** (인식, 기록, 피드) | 95% | GPT Vision 전환으로 기능 완성도 극대화 |
| **안정화 및 버그 수정** | 80% | TestFlight 피드백 반영 중. 인증/성능 관련 개선 중점 |
| **성능 최적화** (레이턴시) | 70% | 이미지 처리 속도는 개선되었으나, 지연 로딩 등 추가 작업 필요 |
| **마케팅/운영 시스템** | 50% | 데이터 수집 및 내부 관리 툴 개발 시작 단계 |

다음 기간에는 성능 최적화(Performance Refactoring)와 더불어, 개발팀 외 인원들이 내부 데이터를 쉽게 관리할 수 있는 Admin 페이지 구현에 착수할 계획입니다.
